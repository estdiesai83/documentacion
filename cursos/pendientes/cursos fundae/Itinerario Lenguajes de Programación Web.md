## 1. Introducción a la programación web

### 1.1 Conceptos básicos de la web y arquitectura cliente-servidor

**Frontend** es la parte del desarrollo que ven los usuarios, la cara que muestra la aplicación.

El **backend** es el “alma”: la lógica, el servidor, la aplicación y la base de datos de la página. 

Y ambas se relacionan a través de las APIs, intercambiando solicitudes del frontend y datos del backend. 

Las redes permiten que las computadoras se comuniquen entre sí, enviando y recibiendo datos entre ellas. Esto hace posible trabajar de forma remota, utilizar las redes sociales, hacer videoconferencias o que participes en este curso.



- **¿Qué es la World Wide Web?**

  La World Wide Web (WWW) es un sistema de **documentos y recursos** web interconectados mediante hipervínculos y accesibles a través de Internet.

  Tim Berners-Lee, un científico de la computación que trabajaba en el CERN, creó la web en 1989. Su objetivo era crear un **espacio de información global** que permitiera a las personas compartir conocimientos y comunicarse independientemente de su ubicación geográfica.

- **Sistema de hipertexto**

  La propuesta inicial de Berners-Lee incluía la creación de un **sistema de hipertexto** que utilizara Internet como medio de transmisión.

- **Evolución de la web**

  La web ha evolucionado desde simples páginas de texto en los primeros días del navegador Mosaic, a las complejas aplicaciones web de hoy en día que permiten interacciones en tiempo real, transacciones seguras, y experiencias multimedia ricas.

  Esta evolución ha sido posible gracias al desarrollo continuo de tecnologías web, estándares y protocolos.



#### Arquitectura cliente-servidor

La arquitectura cliente-servidor es un **marco de diseño de software** en el que las tareas se reparten entre los proveedores de recursos o servicios, conocidos como servidores, y los solicitantes de servicios, conocidos como clientes.

Los clientes realizan solicitudes a los servidores, que luego procesan la solicitud y devuelven la respuesta adecuada. Esta arquitectura es fundamental para el funcionamiento de la web, permitiendo la **comunicación y transferencia de datos** entre sistemas dispersos geográficamente.

La implementación de esta idea dio origen a tres tecnologías fundamentales que forman la base de la web actual: 

- **HTML** (Lenguaje de Marcado de Hipertexto): Diseñado para estructurar y presentar contenido en la web. 
- **URI** (Identificador de Recursos Uniforme): Permite identificar de manera única recursos en la web. 
- **HTTP** (Protocolo de Transferencia de Hipertexto): Define cómo se solicitan y transmiten los recursos entre clientes y servidores.



**Funcionamiento y componentes**

##### Cliente

En el contexto de la web, el cliente suele ser un **navegador web** que envía solicitudes al servidor para acceder a páginas de información. Las páginas web que se reciben se transfieren con el protocolo HTTP. El cliente interpreta el contenido recibido del servidor y lo muestra al usuario. Si lo que se recibe son objetos como vídeo, imágenes, o cualquier otro recurso, el cliente debe activar aplicaciones externas que puedan gestionarlo, como los navegadores.



##### Servidor

El servidor es un **sistema de software o hardware** que responde a las solicitudes del cliente a través de la red. Los servidores web almacenan los recursos web, como archivos HTML, CSS, y JavaScript, y los envían a los clientes según sea necesario.

Funciona de la siguiente manera: si encuentra en su sistema de ficheros el documento HTML solicitado por el cliente, lo envía y cierra la conexión; en caso contrario, envía un **código de error** que cierra la conexión.



La comunicación entre clientes y servidores se realiza utilizando el protocolo **HTTP**. Cuando un usuario escribe una URL en su navegador, este envía una solicitud HTTP al servidor correspondiente. El servidor procesa la solicitud y envía una respuesta HTTP al cliente, que puede ser una página web, un mensaje de error, o cualquier otro recurso web.

**Importancia para profesionales de la programación**
Entender la arquitectura cliente-servidor es crucial para desarrolladores web, ya que proporciona la base para diseñar y construir aplicaciones web eficientes y seguras. Los profesionales deben comprender cómo optimizar las solicitudes y respuestas entre el cliente y el servidor, cómo gestionar las sesiones de usuario, y cómo asegurar la comunicación contra amenazas cibernéticas.



### 1.2 Navegadores web y herramientas de desarrollo

Los navegadores web son **herramientas muy utilizadas** en nuestra vida cotidiana, pues nos permiten el acceso y la navegación por la World Wide Web.

Estos programas interpretan el código de las páginas web (principalmente HTML, CSS, y JavaScript) y lo presentan visualmente al usuario.

**Origen:** 

Tim Berners-Lee creador de la World Wide Web (www) creó el primer navegador con el mismo nombre, más tarde lo llamaron Nexus. En 1993, Marc Andreessen lideró el desarrollo de Mosaic, el primer navegador gráfico, en el NCSA.  

Mosaic hizo que la web fuera más accesible para el público general al soportar imágenes inline con el texto, lo que era una novedad en ese momento. La popularidad de Mosaic llevó al desarrollo de Netscape Navigator en 1994, que dominó el mercado hasta la llegada de Internet Explorer de Microsoft. 

El lanzamiento de Mozilla Firefox en 2004 y de Google Chrome en 2008 marcó el inicio de una nueva era para los navegadores web, que se centraron en la velocidad, la seguridad y el soporte de estándares web. 

Chrome, en particular, jugó un papel crucial en la promoción de innovaciones como las aplicaciones web progresivas (PWA) y la mejora de la experiencia del usuario.

### 1.3 Frontend frente a backend

Frontend y backend son dos componentes críticos que **trabajan en conjunto** para crear aplicaciones web ricas e interactivas.

La distinción entre ambos tipos de desarrollo surgió con la tecnología HTML5, y permite incorporar todo tipo de recursos a una página. Estas dos partes de una aplicación **intercambian información a través de las API**.

Una **API** (interfaz de programación de aplicaciones) permite que diferentes aplicaciones se comuniquen entre sí.

![image-20250211114427635](/home/soa/.config/Typora/typora-user-images/image-20250211114427635.png)

#### Frontend

El desarrollo frontend se refiere a la parte de la aplicación web que **los usuarios ven y con la que interactúan** directamente. Es el rostro visual de la aplicación, donde la usabilidad, el diseño y la experiencia del usuario son primordiales.

**Tecnologías del Frontend** 

- **HTML** (Lenguaje de Marcado de Hipertexto): La columna vertebral de cualquier página web, estructura el contenido y los elementos visuales en la web.
- **CSS** (Hojas de Estilo en Cascada): Define el estilo y la presentación de los documentos HTML, incluyendo colores, diseños y fuentes. 
- **JavaScript** (JS): Añade interactividad a las páginas web, permitiendo la creación de contenido dinámico como formularios en vivo, animaciones y la manipulación de elementos HTML y CSS.

**Frameworks y bibliotecas** 

- **React**: Una biblioteca de JavaScript para construir interfaces de usuario interactivas de manera eficiente. 
- **Angular**: Un framework de desarrollo para construir aplicaciones web dinámicas y de una sola página (SPA). 
- **Vue.js**: Un framework progresivo para construir interfaces de usuario y aplicaciones de una sola página.



#### Backend

El desarrollo backend se centra en la "parte trasera" de la aplicación web, el servidor, la aplicación y la base de datos. Es donde **se procesa la lógica de negocio**, se gestionan las solicitudes de los usuarios, y se almacena y recupera la información.

**Tecnologías del Backend** 

- **Lenguajes de Programación**: Python, Ruby, PHP, Java, y Node.js son comunes en el desarrollo backend, cada uno con sus propias bibliotecas y frameworks. 
- **Bases de Datos**: Sistemas como MySQL, PostgreSQL, MongoDB, y Firebase para almacenar y gestionar datos. 
- **Servidores Web**: Apache y Nginx son ejemplos de software de servidor web que gestionan las solicitudes entrantes de Internet y sirven respuestas a los usuarios. 

**2.2 Arquitectura y diseño del Backend** 

- **APIs** (Interfaces de Programación de Aplicaciones): Permiten la comunicación entre diferentes partes de una aplicación o entre distintas aplicaciones. 
- **Microservicios**: Una arquitectura que divide una aplicación en una colección de servicios más pequeños, lo que mejora la modularidad y facilita la escalabilidad y el mantenimiento.

#### Interacción entre frontend y backend

La interacción entre el frontend y el backend es esencial para el funcionamiento de las aplicaciones web. Se realiza principalmente **a través de API**, donde el frontend hace solicitudes al backend, que procesa estas solicitudes y devuelve los datos necesarios para que el frontend los presente al usuario.

Ya hemos visto que el protocolo HTTP es la base de todos los intercambios de información en la web. 

**Protocolo HTTP** El protocolo HTTP (Hypertext Transfer Protocol) es el fundamento de cualquier transferencia de datos en la web, facilitando la comunicación entre cliente y servidor.

Además, tenemos JSON y AJAX:

- **JSON (JavaScript Object Notation)**

  JSON es un formato ligero de intercambio de datos, fácilmente legible tanto para humanos como para máquinas.

- **AJAX (Asynchronous JavaScript And XML)**

  AJAX permite al frontend realizar solicitudes al backend de manera **asíncrona**, sin necesidad de recargar la página completa.

El cliente y el servidor son dos máquinas, el cliente es quien pide información y el servidor es quien entrega la información que pide el cliente, siempre y cuando el cliente tenga los permisos, es decir el servidor hace la validación de que el cliente tenga los permisos.

Ej: el cliente entra en una web y a trvés de la conexión a internet esta máquina cliente se conecta con el servidor donde este revisa si tienes los permisos para ver lo que estas solicitando por, ejemplo estas solicitando un video de una clase premium, entonces el servidor verá este usuario tiene acceso o no tiene acceso y según eso, el servidor entregará o no entegrará el video.

Cada solicitud que hace el cliente al servidor se llama **petición**, en el desarrollo web tradicional cada petición al servidor recargaba completamente la página, por ejemplo si tu querias cambiar solamente una foto tenías que traerte toda la página completa de nuevo, no solamente esta foto, pero con tecnologías como **AJAX** o **websockets** podemos hacer peticiones de cosas muy específicas y  sólamente cambiar esas cosas en la pantalla haciendo que la web sea mucho mas fluida, más dinámica.

**Historia de HTML**

HTML fue creado por Tim Berners-Lee en 1990 mientras trabajaba en el CERN y fue diseñado para **facilitar el intercambio de información científica y académica entre investigadores.** Desde su creación, HTML ha pasado por varias revisiones y actualizaciones, hasta llegar a HTML5, que es la versión actual y la más avanzada, incorporando soporte para multimedia, gráficos, efectos interactivos y compatibilidad con dispositivos móviles.

HTML es fundamental para el desarrollo web porque proporciona la **estructura básica de todas las páginas web**. Sin HTML, no podríamos organizar el texto, crear enlaces, insertar imágenes o videos, ni realizar muchas otras funciones que damos por sentadas en las páginas web modernas. 



**Estructura básica de una página web**

Cada documento HTML sigue una estructura estándar, que asegura que los navegadores puedan interpretar y mostrar el contenido de manera correcta a los usuarios.

Los **componentes básicos** que lo constituyen: los **elementos** y las **etiquetas HTML**. Estos son los bloques de construcción fundamentales de cualquier página web, permitiendo a los desarrolladores definir la estructura, organizar el contenido y especificar la presentación de la información en el navegador. 

Esta estructura básica incluye varios elementos esenciales que trabajan juntos para definir el encabezado (head) y el cuerpo (body) de la página, entre otros componentes. Aprender a estructurar correctamente un documento HTML es el primer paso crítico para cualquier desarrollador web, ya que establece la base sobre la que se construirán estilos con CSS y se añadirán funcionalidades con JavaScript.

**Definición y sintaxis**

- **Definición**: En HTML, un elemento consiste en una etiqueta de apertura, que contiene el contenido, seguida de una etiqueta de cierre. Las etiquetas son indicadores que le dicen al navegador cómo debe procesar y presentar el contenido entre ellas. Por ejemplo, un elemento de párrafo se representa mediante las etiquetas <p> (apertura) y </p> (cierre), y el texto que se coloca entre estas etiquetas se muestra como un párrafo en la página web. 
- **Sintaxis**: La sintaxis de una etiqueta HTML típica incluye el nombre de la etiqueta encerrado en corchetes angulares (< >). 
  - Cuando una etiqueta requiere un cierre, se utiliza una barra inclinada (/) antes del nombre de la etiqueta en la etiqueta de cierre. 
  - Algunos elementos, conocidos como elementos vacíos (por ejemplo, la etiqueta de imagen <img> y la etiqueta de salto de línea < br>, no contienen contenido y por lo tanto no requieren una etiqueta de cierre. 

**Elementos de apertura y cierre** 

La mayoría de los elementos HTML se componen de una etiqueta de apertura y una etiqueta de cierre, con el contenido relevante ubicado entre ellas. La etiqueta de apertura marca el inicio del elemento, mientras que la etiqueta de cierre señala el final. Este par de etiquetas define el alcance del elemento en el documento. Por ejemplo, en < h1> encabezado principal </h1>, <h1>  es la etiqueta de apertura y </h1> es la etiqueta de cierre, y juntas envuelven el texto del encabezado principal. 

En contraste, los elementos vacíos o auto-cerrables no encierran contenido y por lo tanto no requieren una etiqueta de cierre. Estos elementos a menudo incluyen atributos para especificar su comportamiento o presentación. Un ejemplo clásico es la etiqueta de imagen <img>), que utiliza atributos como src (para especificar la URL de la imagen) y alt (para proporcionar texto alternativo en caso de que la imagen no pueda mostrarse).



**< !DOCTYPE html>** 

Define el tipo de documento y la versión de HTML. Ayuda al navegador a entender qué versión de HTML está siendo utilizada para interpretar correctamente el contenido de la página. En HTML5, la última versión de HTML, esta declaración es simplificada a: <!DOCTYPE html>  

**Elemento  con los atributos lang**

Después, pondrías el elemento  <html>, actúa como el contenedor raiz de todo el contenido de la página.

El atributo lang dentro del elemento  es importante para definir el idioma principal del documento, lo que ayuda a los motores de búsqueda y las tecnologías de asistencia a entender y procesar el texto adecuadamente. Por ejemplo,  indica que el idioma principal del documento es el inglés

**Secciones <head> y <body>** 

Dentro del elemento , la estructura se divide en dos secciones principales: <head>  y <body>. 

- Sección <head> : Esta sección contiene metadatos sobre el documento, que no son visibles directamente en la ventana del navegador pero son esenciales para la definición de la página y la comunicación con los navegadores y motores de búsqueda. Elementos comunes dentro del <head> incluyen <title>, que define el título de la página mostrado en la pestaña del navegador. <meta> tags, que pueden especificar la codificación de caracteres, la descripción de la página y las palabras clave; enlaces a hojas de estilo externas (<link>) y scripts (<script>).
- Sección <body>: Aquí es donde se coloca el contenido real que los usuarios verán y con el que interactuarán en el navegador, como texto, imágenes, videos, enlaces, formularios y más. Todo lo que se incluye en esta sección se muestra en la ventana del navegador.



**Metadatos en el <head>**

La sección  de un documento HTML es una parte esencial pero invisible de la estructura de una página web. Contiene metadatos, o datos sobre los datos, que proporcionan información crucial sobre el documento HTML. 

Estos metadatos no se muestran directamente en la parte visible de la página web, pero influyen en cómo los navegadores y los motores de búsqueda interpretan y procesan la página.

**¿Qué son los metadatos y qué papel cumplen?** 

Los metadatos en HTML son un conjunto de datos que proporcionan información sobre el contenido de una página web. Estos datos ayudan a los navegadores a entender cómo deben mostrar el contenido, permiten a los motores de búsqueda indexar la página de manera más efectiva y pueden incluir instrucciones para controlar el comportamiento de la página o informar a los usuarios sobre su contenido. En esencia, los metadatos mejoran la usabilidad, accesibilidad y SEO (optimización para motores de búsqueda) de una página web. Metadatos clave en el  <head>

-  **< title>**: El elemento <title> define el título de la página web, que se muestra en la barra de título del navegador y en los resultados de búsqueda. Es un factor crucial para el SEO y la usabilidad, ya que
  ofrece a los usuarios una rápida orientación sobre el contenido de la página.
- **<meta>**: Este elemento se utiliza para especificar metadatos que no pueden ser definidos por otros elementos HTML. Los atributos comunes incluyen **name** y **content**, donde name puede ser, por ejemplo, description (ofreciendo una breve descripción de la página) o keywords (definiendo una lista de palabras clave relevantes para el contenido de la página), y content contiene el valor de estos atributos. Además,  especifica la codificación de caracteres para la página web, asegurando la correcta visualización de texto.
- **<link>**: Utilizado principalmente para vincular hojas de estilo externas (CSS) con el documento HTML. Por ejemplo,  <link rel="stylesheet" href="styles.css" enlaza un archivo CSS que define la presentación visual de la página.

- < **style**>: Permite incluir CSS directamente dentro del documento HTML para estilizar elementos de la página. Aunque es más común utilizar hojas de estilo externas para mantener la separación entre contenido y presentación, este elemento puede ser útil para estilos específicos de una sola página. 
- **< script>:** Utilizado para incluir o referenciar código JavaScript, que añade interactividad y funciones dinámicas a la página web. Los scripts pueden incluirse directamente entre las etiquetas <script> o referenciarse a través del atributo src que apunta a un archivo Javascript externo.

Los metadatos en la sección  juegan un papel vital en la funcionalidad, presentación y optimización de una página web. Al proporcionar información esencial a los navegadores y a los motores de búsqueda, estos elementos aseguran que la página web no solo funcione como se espera, sino que también sea accesible y visible para el público objetivo.

### 2.2 Etiquetas y atributos comunes.

Las etiquetas HTML son los bloques de construcción básicos de la web. Funcionan como **contenedores que dicen al navegador cómo debe presentar el contenido que encierran**. Las etiquetas no trabajan solas. A menudo, van acompañadas de atributos, pequeños fragmentos de información que modifican la función de una etiqueta o añaden funcionalidades adicionales, como enlaces que nos llevan a otra página o imágenes que embellecen y explican nuestro contenido.

Una etiqueta HTML es una instrucción para el navegador, un guía que le dice cómo debe presentar el contenido encerrado dentro de ella. Las etiquetas son esenciales para la estructura y accesibilidad del contenido.

Imagina que cada etiqueta es como un ladrillo en la construcción de un edificio: cada una tiene su lugar y propósito, contribuyendo a la estructura general y la estética del sitio.

#### Etiquetas comunes en HTML

Estas etiquetas son las herramientas básicas en el arsenal de todo desarrollador web, permitiéndonos estructurar el texto, enlazar recursos, incrustar imágenes y organizar la información de manera coherente y accesible.

**Etiquetas de apertura y cierre** 

Las etiquetas HTML generalmente vienen en pares: 

- Etiqueta de apertura que inicia el elemento (<etiqueta>) y una etiqueta de cierre que lo finaliza (</etiqueta>). Esta simetría asegura que el contenido esté claramente delimitado, permitiendo al navegador saber exactamente dónde comienza y termina cada elemento. 
- Etiquetas auto-cerradas (<etiqueta/>), que se encargan de tareas específicas, como incrustar una imagen o insertar un salto de línea, sin necesidad de encerrar contenido entre una apertura y un cierre.



**Ejemplos en acción** 

Para visualizar cómo funcionan estas etiquetas en la práctica, consideremos algunos ejemplos sencillos: 

Un párrafo de texto se encierra entre las etiquetas de apertura y cierre <p> y </p>

```html
<p>Este es un párrafo de ejemplo</p>
```

Una imagen se inserta mediante una etiqueta auto-cerrada <img>, utilizando atributos para especificar la fuente de la imagen y una descripción alternativa 

```html
<img src="ruta/a/la/imagen.jpg" alt="Descripción de la imagen">
```

 Son los faros que guían a los usuarios a través de nuestra página, indicando los títulos y sub-títulos. Desde <h1> el más importante, hasta <h6> el menos prominente, estos estructuran el contenido jeráquicamente

```html
<h1> Este es un encabezado principal</h1>
```

```html
<p> Es <strong>importante</strong> enfatizar esto, no lo olvides.</p>
<p> Para resaltar algo con <em>delicadeza</em>, usa itálica</p>
```

span es un elemento contenedor en línea genérico. Las etiquetas span generalmente envuelven secciones de texto con fines de estilo o para agregar atributos a una sección de texto sin crear una nueva línea de contenido.

La web es una red interconectada gracias a los enlaces, y la etiqueta **<a>** es el puente que conecta sus múltiples nodos: 

- **Uso básico**: Con el atributo:

  - **href**: definimos el destino del enlace. 
  - **target**: controla cómo se abrirá el enlace 
    - **self**: se abre en la misma ventana en la que se encuentra el documento que contiene el enlace. Es el valor por defecto.
    - **_blank**: se abre en una nueva ventana o pestaña. En los navegadores actualizados se abre en una nueva pestaña ya que es la recomendación a partir de HTML 5. Si queremos que se abra en una nueva ventana deberemos hacerlo con programación web (javascript, php, ...)
    - **_parent** se abre en el **frameset** padre. Este valor tiene sentido cuando estamos utilizando **frameset**, algo que ya no está recomendado.
    - **_top** se abre en la pantalla completa, fuera del **frameset**. Sólo cuando estamos utilizando frameset.
    - **nombreframe** se abre en el nombre del frame.

- **title:** Proporciona información adicional sobre un elemento, generalmente mostrada como una "tooltip" cuando el cursor pasa sobre él.

#### Contenedores

La estructuración del contenido no es simplemente una cuestión de estética; es fundamental para la accesibilidad, la usabilidad y el SEO. Los contenedores son nuestros pinceles para delinear espacios, agrupar elementos y aplicar estilos con precisión.

En el lienzo de la web, los contenedores son nuestros pinceles para delinear espacios, agrupar elementos y aplicar estilos con precisión. Las etiquetas <div> y <span> son los artistas versátiles en esta tarea,
permitiéndonos pintar estructuras y patrones en nuestra página.

**< div>:**

El <div> es como un gran contenedor de almacenamiento, perfecto para agrupar bloques de contenido y elementos de diseño. Su naturaleza neutra lo hace ideal para ser estilizado con CSS, convirtiéndolo en el pilar de la estructura y el diseño web. Puedes pensar en <div> como un bloque amplio y flaxible que puede contener otros elementos, como texto, imágenes o incluso otros divs.

Es ideal para dividir tu página en secciones como encabezados, navegación, contenido principal y pie de página.

```html
<div class="seccion-menu">
	<!--Contenido del menú aqui-->
</div>
```

**< span>:** (detalles y estilo fino)

Mientras que <div> se ocupa de los grandes esquemas, <span> se especializa en los detalles. Utilizado para agrupar contenido en línea dentro de otros elementos, <span> nos permite aplicar estilos a fragmentos de texto o pequeños componentes sin alterar el flujo del documento.

A diferencia del **div, ** el **span** es un elemento en línea, lo que significa que no agrega saltos de línea automáticos. Puedes envolver una palabra, una frase o incluso una letra con la etiqueta span y luego aplicar estilos, como cambios de color, subrayado o resaltado, mediante CSS.

**Span** es útil cuando deseas resaltar o dar formato a partes individuales de un texto sin afectar al resto del contenido.

```html
<p>Este es un <span class="texto-destacado">texto destacado</span>dentro de un parrafo</p>
```



**Diferencias entre div y span**

**div** es un elemento de bloque, por lo que crea un salto de línea automático antes y despues de él, ideal para dividir y organizar secciones de tu página web.

Se utiliza para aplicar estilos a nivel superior a un grupo de elementos, como fondos, bordes o márgenes.

**span** se ajusta al flujo del texto o del elemento que lo contiene, sin crear divisiones visuales en el diseño.

Se utiliza para aplicar estilos más detallados, como cambios de color, tamaño de fuente o subrayado, a partes específicas de un texto o de elementos en línea. 

Hay que evitar un uso excesivo de divs sin una razón clara y evita envolver grandes bloques de texto en spans incecesariamente.

### 2.3 Uso de imágenes, enlaces y listas.

Las imágenes capturan la atención y comunican miles de palabras en un instante. Los enlaces coectan el mundo de la web, guiando a los visitantes a través de un viaje de descubrimiento e interacción. 

Las listas, son los pilares del orden, organizando la información de manera que sea fácilmente digerible y accesible.

#### La etiqueta < img>

Es el modo que HTML ofrece para incluir imágenes en tu página web. Características:

- Actúa como un contenedor invisible que no necesita cierre, su propósito es referenciar y mostrar una imagen alojada en otra parte, no contener otro contenido.
- Su simplicidad es engañosa, pues a través de sus atributos se puede incorporar mucha variedad. 

La etiqueta <img> se apoya en varios **atributos esenciales**, cada uno desempeñando un papel único en la presentación y accesibilidad de las imágenes.

- **src (source)**:

  Es el pilar sobre el cual se construye cada <img>. Proporciona la dirección exacta, o URL, donde se aloja la imagen que deseas mostrar. Sin este atributo, la imágen es como un marco vacío. Con este atributo, le indicas al navegador donde encontrar la imagen que debe recuperar y mostrar en tu página.

  ```html
  <img src="ruta/a/la/imagen.jpg">
  ```

  

- **alt (alternative text Texto alternativo)**:

  El texto alternativo de cada imagen en la web, es su voz cuando la visualización directa no es posible, ya sea por usuarios que dependen de lectores de pantalla, conexiones lentas que impiden la carga de imágenes, o simplemente imágenes que no se pueden mostrar debido a a errores, el atributo **alt** asegura que el contenido y la función de la imagen aún se comuniquen.

  Siempre proporciona un texto **alt** significativo para tus imágenes. No solo es una cuestión de accesibilidad, sino que también mejora el SEO de tu página, permitiendo que los motores de búsqueda comprendan el contexto de las imágenes utilizadas .

  ```html
  <img src="ruta/a/la/imagen.jpg" alt="Descripción de la imagen">
  ```

  

- **Optimización de imágenes:**

  El tamaño y la calidad de tus imágenes deben equilibrarse para mantener rápidos tiempos de carga sin sacrificar la estética.

  - **width y heigth**

    Estos atributos definen el ancho y el alto en la página, permitiéndote controlar su tamaño de visualización. Aunque son opcionales, su uso puede ser estratégico para mantener la disposición visual de tu página mientras las imágenes se están cargando, y para asegurar que las imágenes se muestren en las dimensiones deseadas sin necesidad de recurrir a ajustes de CSS o a la alteración de la imagen original.

    ```html
    <img src="imagen.jpg" alt="Descripción de la imagen" width="500" heigth="300">
    ```

    

  **Herramientas de optimización en línea** pueden ayudarte a comprimir imágenes sin perder calidad visible, para mantener rápidos tiempos de carga sin sacrificar la estética..

- **Elección del formato:**

  Selecciona el formato de imagen más adecuado para tus necesidades, considerando la **calidad**, el **tamaño** del archivo y la **funcionalidad** (como la animación o la transparencia).



#### Creación de enlaces

Los enlaces son el tejido conectivo de la World Wide Web, uniendo ideas, páginas y recursos en una red interconectada de información. La etiqueta <a> es el pilar de la navegación web. 



La etiqueta <a>, abreviatura de "ancla" permite crear enlaces que guían a los usuarios a:

- diferentes partes de nuestra página o a otros sitios web,
- acciones, como abrir una aplicación de correo electrónico.

La verdadera magia de los enlaces reside en su capacidad para ser moldeados y adaptados a nuestras necesidades específicas, gracias a una serie de atributos clave.

Crear enlaces accesibles y fáciles de usar es crucial para una web inclusiva.

**Atributos de enlace:**

- **href (Hypertext Reference):**

  Especifica el destino del enlace, ya sea una URL externa que lleva a otro sitio o una ruta interna que navega a otra sección de la misma página.

  ```html
  <a href="https://www.ejemplo.com">Visita Ejemplo</a>
  ```

  En cuanto a los enlaces dentro de la página o internos, se llaman **Anclas.**

  Las anclas permiten crear enlaces de navegación interna que llevan a los usuarios a puntos específicos dentro de la misma página, mejorando la experiencia del usuario en páginas largas o complejas.

  ```html
  <a href="#seccion1">Ir a la sección 1</a>
  ...
  <div id="seccion1">
       Contenido de la seccion1
  </div>
  ```

- **target**

  Determina como se abrirá el enlace en la ventana del navegador.

  - **_blank**: abre el enlace en una nueva pestaña o ventana, fomentando una navegación que mantiene a los usuarios en tu página original.

    ```html
    <a href="https://www.ejemplo.com" target="_blank">Visita Ejemplo en una nueva pestaña</a>
    ```

  - **title**: Proporciona información adicional sobre el enlace, generalmente mostrada como una "tooltip" al pasar el cursos sobre el enlace. Mejora la accesibilidad y proporciona contexto adicional a los usuarios.

    ```html
    <a href="https://www.ejemplo.com" target="_blank" title="Aprende mas sobre Ejemplo">Visita Ejemplo en una nueva pestaña</a>
    ```

  - **rel**: Especifica la relación entre la página actual y el destino del enlace, como nofollow para indicar a los motores de búsqueda que no sigan ese enlace o noopener para mejorar la seguridad al abrir enlaces en nuevas pestañas.

    ```html
    <a href="https://www.ejemplo.com" target="_blank" rel="noopener noreferrer">Visita Ejemplo en una nueva pestaña</a>
    ```

​	

**Aspectos de usabilidad y accesibilidad** 

Crear enlaces accesibles y fáciles de usar es crucial para una web inclusiva*.

*Una web inclusiva es un sitio web diseñado y desarrollado para ser accesible y usable por la mayor cantidad de personas posible, incluyendo aquellas con discapacidades. 

Usa textos de enlace descriptivos que informen a los usuarios sobre el destino del enlace, evita el texto genérico como "haz clic aquí", y asegúrate de que los enlaces sean fácilmente identificables con estilos distintivos (el texto subrayado y con el clásico color azul). La accesibilidad no solo beneficia a los usuarios con discapacidades, sino que mejora la experiencia de todos los usuarios y ayuda a los motores de búsqueda a entender mejor tu sitio.

**Tipos de Listas:**

  ```html

      <li>Primer paso</li>
  ```

  Ideales para listas donde el orden de los ítems no es relevante, como

### 2.4 Formularios y elementos de entrada.

Actúan como **interfaces para que los usuarios introduzcan y envíen datos**, que luego pueden ser procesados por el servidor o utilizados de manera dinámica en la propia página. 

#### Estructura básica de un formulario

Colección de campos de entrada que permiten a los usuarios ingresar datos. Estos datos pueden ser texto, números, selecciones de opciones, marcas de verificación y más. 

Una vez que un usuario completa y envía un formulario, la información ingresada se envía al servidor para su procesamiento. 

Los formularios son fundamentales para tareas comunes en la web, como: 

- **Registros y Logins**: Donde los usuarios ingresan datos personales para acceder a áreas restringidas o servicios personalizados. 

- **Búsquedas**: Permitiendo a los usuarios introducir términos de búsqueda para encontrar información específica.
- **Envíos**: Como la publicación de comentarios, la carga de archivos, o la realización de pedidos en línea.

La habilidad para diseñar formularios eficientes y amigables es crucial para la experiencia del usuario, ya que una interfaz de formulario bien diseñada puede fomentar la interacción y mejorar la percepción general de un sitio web.

La **etiqueta <form>** es el contenedor para todos los elementos de un formulario. Define el espacio donde los campos de entrada y los botones de envío se agrupan y establece cómo se enviarán los datos.

Dentro del HTML de un formulario, las **etiquetas <label> y <input>** suelen usarse juntas para crear una interfaz de usuario clara y accesible.

- La etiqueta <label> proporciona una descripción o identificación del campo de entrada que se va a llenar.
- La etiqueta <input> define el propio campo de entrada donde los usuarios pueden introducir sus datos.

Los **elementos de entrada** son las herramientas mediante las cuales los usuarios interactúan con los formularios. Cada tipo de elemento de entrada está diseñado para recopilar un tipo específico de dato, desde textos simples hasta selecciones complejas.

A medida que los formularios se vuelven más complejos y recopilan una variedad más amplia de información, es esencial utilizar **controles avanzados y técnicas de estructuración** para mejorar la experiencia del usuario y la coherencia de los datos.

Crear formularios **accesibles y bien diseñados** es esencial para ofrecer una experiencia de usuario inclusiva y efectiva.

**Asociación entre label y input** 

Hay dos maneras principales de asociar una etiqueta <label> con un campo <input>:

  ```html

      <input type="text" name="nombre">
  </label>
  ```

### 2.5 Semántica y accesibilidad en HTML.

La semántica en HTML se utiliza para formatear el contenido y para **transmitir el significado y la estructura de ese contenido**.

Una buena semántica mejora la accesibilidad, refuerza el SEO y facilita la mantenibilidad del código.

La accesibilidad se asegura de que nuestras páginas web sean utilizables por la mayor cantidad posible de personas, incluyendo aquellas con diversas discapacidades.

Juntos, estos conceptos fomentan una web más inclusiva y comprensible.

**Posibilidades del HTML semántico:**

La diferencia entre las etiquetas HTML "tradicionales" y las "semánticas" radica principalmente en su propósito y significado dentro de la estructura del documento. 

Las etiquetas tradicionales que ya conoces, como **<div>  y <span>** , son contenedores genéricos. No proporcionan información sobre el tipo de contenido que encierran. Su principal función es estructurar la presentación del contenido, a menudo en combinación con atributos de clase o ID para aplicar estilos CSS específicos.

```html
<div class="header">
<div class="navigation">
<!-- Enlaces de navegación aquí --> </div>
</div>
<div class="mainContent">
<div class="article">
<!-- Contenido del artículo aquí -->
</div>
</div>
<div class="footer">
<!-- Contenido del pie de página aquí →-->
</div>
```

Observa este ejemplo. Las etiquetas **<div>** se utilizan para crear la estructura de la página, pero no comunican ningún significado sobre el contenido que contienen. Los nombres de las clases sugieren su propósito, pero solo son claros para los humanos que leen el código, no para las máquinas. ¿Cómo sería con el modelo HTML semántico?

[Cambia el código por este otro]

```html

    <nav>
    <!-- Enlaces de navegación aquí -->
    </nav>
</header>
<section>
    <article>
    <!-- Contenido del artículo aquí -->
    </article>
</section>
<footer>
<!-- Contenido del pie de página aquí -->
</footer>
```

Aquí te lo muestro. Las etiquetas semánticas, introducidas con HTML5, como <header>, <nav>, <article>, <section> y <footer>, indican explícitamente la función y el tipo de contenido que encierran. 

Estas etiquetas mejoran la accesibilidad y la legibilidad del código, tanto para los desarrolladores como para las tecnologías asistivas y los motores de búsqueda. También enfatizan el significado del contenido, lo que contribuye a una mejor experiencia de usuario y SEO.

El **SEO**, o Search Engine Optimization, es la técnica que usamos para mejorar la posición de una web en las búsquedas de los navegadores, principalmente en Chrome de Google. Es un elemento fundamental de la industria del márketing digital.

En el ejemplo se utiliza etiquetas semánticas que definen claramente las diferentes secciones de la página: 

- el área de navegación (<nav> dentro de <header> ), 
- el contenido principal (<article> dentro de <section>)
- el pie de página (<footer>). 

Cada etiqueta describe el propósito del contenido que contiene, lo que facilita su comprensión por parte de las máquinas y mejora la accesibilidad para los usuarios.

- **< header>** representa un grupo de ayudas introductorias o de navegación. Puede contener cosas como títulos de sección, enlaces de navegación o logotipos. Situado en la parte superior de la página, actúa como la "cara" del sitio web.
- **< footer>** define el pie de página de una sección o documento, donde se suelen colocar los datos de contacto, derechos de autor y enlaces a documentos legales.
- **< nav>** contiene los principales bloques de navegación del sitio, como menús o índices. Puede ir dentro del <header> o como un elemento independiente, <nav> encierra los enlaces de navegación principal, ayudando a los usuarios a moverse por el sitio web.
- **< main>** Contiene el contenido principal de la página, diferenciándlo claramente del contenido peroférico en <aside> o la información en el <footer>
- **< article>** encapsula un contenido independiente, que podría distribuirse por si solo, podría existir por si mismo fuera del sitio web, como un post de blog o un artículo de noticas.
- **< section>** define una sección del documento con un tema relacionado, agrupando contenido y subsecciones con un hilo temático común. Agrupa contenido temáticamente relacionado dentro del <main>, cada <section> puede contener su propio <header>, <footer>, y una estructura interna.
- **< aside>** se utiliza para contenido tangencialmento relacionado con el contenido principal, como barras laterales, publicidad o listas de enlaces.

**Principios de Accesibilidad Web:**

Las Pautas de Accesibilidad para el Contenido Web (WCAG) son desarrolladas por el W3C y constituyen el estándar de facto para la accesibilidad web. Estas pautas se basan en cuatro principios fundamentales, conocidos por el acónimo **POUR:**

- **Perceptible:** La información y los componentes de la interfaz de usuarios deben presentarse de manera que todos los usuarios puedan percibirlos, independientemente de sus capacidades sensoriales. Esto incluye proporcionar alternativas textuales para el contenido no textutal, como imágenes, y garantizar que la información sea visible y audible para todos.
- **Operable:** Los componente de la interfaz y la navegación deben ser operables para todos. Esto significa que el sitio web debe ser navegable y utilizable con diversas herramientas, como teclados y lectores de pantalla, y que debe evitar diseños que puedan provocar convulsiones o reacciones físicas.
- **Comprensible:** La información y la operación de la interfaz de usuario deben ser comprensibles. Esto implica que el contenido debe ser legible y entendible, y que las páginas web deben aparecer y operar de manera predecible.
- **Robusto:** El contenido debe ser lo suficientemente robusco como para ser interpretado de manera confiable por una amplia variedad de agentes de usuario, incluidas las tecnologías asistivas. Esto significa utilizar un marcado estándar y asegurar la compatibilidad entre navegadores y dispositivos.



**Roles ARIA y Atributos**

ARIA (Accessible Rich Interner Applications) es un conjunto de atributos especiales que se pueden agregar al HTML para mejorar la accesibilidad, especialmente en aplicaciones web complejas que utilizan elementos dinámicos y controles avanzados de la interfaz de usuarios que no son semánticamente representados por las etiquetas HTML estándar.

**Roles ARIA:**

Definene el tipo de elemento de la interfaz de usuario o de la estructura de la página. Por ejemplo, role="navigation" indica que el elemento contiene enlaces de navegación principal dentro de la página.

```html
<nav role="navigation">
	<!-- Enlaces de navegación aqui -->
</nav>
```

**Atributos ARIA:**

Proporcionan propiedades adicionales a los elementos para mejorar la accesibilidad. Estos pueden incluir:

- estados: aria-expanded, aria-selected
- propiedades: aria-label, aria-labelledby (describen las relaciones o características de los elementos).

El uso adecuado de ARIA puede mejorar significativamente la accesibilidad de los elementos interactivos y dinámicos, pero debe utilizarse con precaución y sólo cuando los elementos HTML semánticos no puedan lograr el mismo resultado.

```html
<button aria-expanded="false" aria-controls="menu">Menu</button>
```

Para más información:

https://developer.mozilla.org/es/docs/Web/Accessibility/ARIA/Attributes



La accesibilidad no es un 'extra' o una consideración posterior en el desarrollo web; es una parte integral de un diseño inclusivo y ético. Al seguir los pricipios de WCAG y utilizar herramientas como ARIA cuando sea necesario, podemos asegurar que nuestros sitios web sean accesibles y utilizables para todos.

El uso adecuado de ARIA puede mejorar significativamente la accesibilidad de los elementos interactivos y dinámicos, pero debe utilizarse con precaución y solo cuando los elementos HTML semánticos no puedan lograr el mismo resultado.



## 3. CSS Estilos para la presentacion visual de la página web.

Mientras que HTML se encarga de la estructura y el contenido, CSS se ocupa de la presentación visual,.

CSS (Cascading Style Sheets), es un lenguaje de diseño utilizado para controlar la presentación visual y el estilo de un sitio web. Comprender la sintaxis y la estructura de CSS es esencial para aplicar estilos eficazmente y mantener tus hojas de estilo organizadas y fáciles de leer.

**División de tareas: HTML y CSS**

En el desarrollo web, HTML y CSS trabajan mano a mano para construir páginas web, pero cada uno tiene un rol específico para lograr diseños webs efectivos y mantenibles.

- **HTML (Hypertext Markup Language):** es el esqueleto de la página web. Se encarga de estructurar y organizar el contenido, definiendo elementos como textos, enlaces, imágenes y otros componentes que componen la página. HTML utiliza etiquetas para marcar diferentes tipos de contenido, estableciendo la base sobre la cual se aplicarán los estilos.
- **CSS (Cascading Style Sheets):** es como el traje a medida que se adapta al esqueleto proporcionado por HTML. CSS se encarga del aspecto visual y la presentación de la página, permitiendo a los desarrolladores controlar el layout, los colores, las fuentes, los espacios,...

**Formas de enlazar CSS**

En principio, tenemos **tres** formas diferentes de hacerlo, siendo la primera la más común y la última la menos habitual:

| Mediante...                        | Descripción                                                  |
| ---------------------------------- | ------------------------------------------------------------ |
| Etiqueta `<link rel="stylesheet">` | **Archivo CSS externo**: El código se escribe en un archivo `.css` a parte. |
| Etiqueta `<style>`                 | **Bloque de estilos**: El código se escribe en una etiqueta `<style>` en el documento HTML. |
| Atributo HTML `style="..."`        | **Estilos en línea**: El código se escribe en un atributo HTML `style` en una etiqueta. |

Veamos las ventajas e inconvenientes de cada una de ellas detalladamente:

#### [Archivo CSS externo](https://lenguajecss.com/css/introduccion/como-usar-css/#archivo-css-externo)

En la cabecera de nuestro documento HTML, más concretamente en el bloque `<head></head>`, podemos incluir una etiqueta `<link>` con la que establecemos una relación entre el **documento HTML actual** y el archivo `.css` que indicamos en el atributo `href`.

Veamos el código de nuestro `index.html`:

```html
<link rel="stylesheet" href="index.css" />
```



De esta forma, los navegadores sabrán que deben aplicar los estilos que se encuentren en el archivo `index.css` que está junto al documento `index.html` actual. Se aconseja escribir esta línea lo antes posible (*sobre todo, antes de los scripts*), obligando así al navegador a aplicar los estilos cuanto antes y eliminar la **falsa percepción visual** de que la página está en blanco y no ha sido cargada por completo.

> > > Antiguamente se utiliza un atributo `type="text/css"` que ya no es necesario en HTML5. Se puede indicar para mantener retrocompatibilidad con navegadores muy antiguos, pero actualmente se puede omitir de forma segura.

#### [Bloque de estilos](https://lenguajecss.com/css/introduccion/como-usar-css/#bloque-de-estilos)

Otra de las formas que existen para incluir estilos CSS en nuestra página es la de añadirlos directamente en el documento HTML, a través de una etiqueta `<style>` que contendrá el código CSS:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>Título de la página</title>
    <style>
      div {
        background: hotpink;
        color: white;
      }
    </style>
  </head>
  ...
</html>
```



Este sistema puede servirnos en ciertos casos particulares, pero hay que darle prioridad al método anterior (*CSS externo*), ya que incluyendo el código CSS en el interior del archivo HTML arruinamos la posibilidad de tener el código CSS en un documento a parte, pudiendo **reutilizarlo y enlazarlo** desde otros documentos HTML mediante la etiqueta `<link>`.

> Aunque no es obligatorio, es muy común que las etiquetas `<style>` se encuentren en la cabecera `<head>` del documento HTML, ya que antiguamente era la única forma de hacerlo.

#### [Estilos en línea](https://lenguajecss.com/css/introduccion/como-usar-css/#estilos-en-línea)

Por último, la tercera forma de aplicar estilos en un documento HTML es el conocido como **estilos en línea**. Se trata de hacerlo directamente, a través del atributo `style` de la propia etiqueta donde queramos aplicar el estilo, colocando ahí las propiedades CSS (*que pueden separarse por `;`*):

```html
<p>¡Hola <span style="color: red; padding: 8px">amigo lector</span>!</p>
```



De la misma forma que en el método anterior, con la etiqueta `<style>`, se recomienda no utilizar este método salvo en casos muy específicos y justificados, ya que los estilos se asocian a la etiqueta HTML en cuestión y no pueden reutilizarse. Es por eso, que se suele considerar una mala práctica por muchos diseñadores cuando la sobreutilizas (*sin una razón de peso*). Sin embargo, es una excelente forma de inyectar o incluir [variables CSS](https://lenguajecss.com/css/cascada-css/css-custom-properties/) en una etiqueta y sus etiquetas hijas.

> En el artículo [Estilos en línea](https://lenguajecss.com/cascada-css/estructura/inline-styles/) profundizamos un poco más sobre esta forma de utilizar estilos CSS.



### 3.1. Sintaxis básica de CSS.

En primer lugar, recuerda que debemos tener el documento `.css` enlazado desde nuestro documento `.html`, preferiblemente desde una etiqueta `<link rel="stylesheet">`. En su atributo `href` colocaremos el nombre del documento `.css` que contiene los estilos:

```html
<!DOCTYPE html>
<html>
<head>
  <title>Título de página</title>
  <link rel="stylesheet" href="index.css" />
</head>
<body>
  <div class="page">
    <h1>Título de la página.</h1>
    <p>Me encanta Manz.dev. Lo veo en Twitch o Youtube todos los días.</p>
  </div>
</body>
</html>
```



Bien, ya tenemos nuestro archivo `.html` conectado con nuestro archivo `.css`. Ahora todo el código CSS que coloquemos en nuestro `index.css` se aplicará al HTML y se verá reflejado en el navegador.

### [Sintaxis CSS](https://lenguajecss.com/css/introduccion/estructura-de-css/#sintaxis-css)

Vamos a centrarnos ahora en el contenido de nuestro documento `index.css`. El **código CSS** de dicho documento, se basa en una serie de conceptos que debemos tener claros antes de continuar. Demos un repaso a cada uno de ellos:

| Concepto       | Descripción                                                  |
| -------------- | ------------------------------------------------------------ |
| **Selector**   | Elemento o elementos del documento que vamos a **seleccionar** para aplicarle un estilo concreto. |
| **Propiedad**  | **Característica principal** que vamos a definir con el valor indicado a continuación. |
| **Valor**      | Cada **propiedad** tiene una serie de valores concretos asignables, con los que tendrá uno u otro comportamiento. |
| **Comentario** | Fragmento de texto entre `/*` y `*/` con anotaciones o aclaraciones para el desarrollador (el navegador los ignorará). |
| **Regla**      | Par de **propiedad** y su respectivo **valor** asociado.     |

Así pues, tras aprender estos conceptos, la **estructura** de un documento `.css` se basaría en organizar el código siguiendo el formato del siguiente bloque:

![Sintaxis visual](https://lenguajecss.com/css/introduccion/estructura-de-css/sintaxis-visual.png)

Vamos a verlo con un ejemplo para afianzar conceptos:

html:

```html
<div class="page">
  <h1>Título de la página.</h1>
  <p>Me encanta Manz.dev. Lo veo en Twitch o Youtube todos los días.</p>
</div>
```

css:

```css
p {
  color: red;   /* Color de texto */
}
```

Resultado:

En este caso, estamos seleccionando todas las etiquetas `<p>` del documento HTML (*en este ejemplo es una sola, pero si existieran más se aplicaría a todas*), y les aplicaremos las reglas que contenga, en este caso una regla que define el **color de texto rojo**. Cada una de estas reglas se terminará con el carácter **punto y coma** (`;`), seguido de la siguiente regla. El último punto y coma es opcional y se puede omitir si se desea.

> **Ojo**: Como hemos visto, se pueden incluir comentarios entre los caracteres `/*` y `*/`, los cuales serán ignorados por el navegador. Estos suelen servir para añadir notas o aclaraciones dirigidas a humanos.

### [Documento CSS](https://lenguajecss.com/css/introduccion/estructura-de-css/#documento-css)

Sin embargo, esto es sólo un ejemplo muy sencillo. Los **documentos CSS** se forman por cientos de reglas de este estilo, con diferentes selectores, propiedades y valores. Habitualmente, cada bloque declarado suele incluir múltiples reglas (*y no sólo una como en el ejemplo anterior*):

![Documento CSS](https://lenguajecss.com/css/introduccion/estructura-de-css/documento-css.png)

Ten en cuenta que, a medida que escribimos **código CSS**, este se va haciendo **más grande** y más difícil de controlar y mantener. Para intentar evitar perder el control del mismo, hay que seguir una serie de buenas prácticas al escribir código, que aunque no son necesarias para que funcione en el navegador, los desarrolladores consideramos obligatorias para que sea más fácil de leer:

- Escribe **una regla por línea**. Será mucho más fácil de leer y modificar.

- Usa la **indentación**. Tras escribir el carácter `{` al empezar un bloque de reglas, las propiedades se separan hacia la derecha. Esto hace que sea más fácil de leer y se considera una buena práctica de desarrollo y de programación en general.

- El **último punto y coma** de un bloque de reglas es opcional. Sin embargo, se suele aconsejar escribirlo para mantener una coherencia y evitar problemas o descuidos al modificar posteriormente el código.

  

#### Reglas de CSS

CSS se presenta y aplica a través de reglas de estilo. Cada regla de CSS está diseñada para controlar la presentación de uno o más elementos HTML en la página web.

Una **regla de CSS** se compone de dos partes principales:

- **Selector**: Especifica el elemento HTML al que se aplicarán los estilos. Puede ser el nombre de una etiqueta HTML, una clase, un ID u otros selectores más avanzados. Cuando un selector coincide con un elemento en el documento HTML, las propiedades CSS definidas en la regla de estilo se aplican a ese elemento.
- **Bloque de declaración:** Contiene una o más declaraciones separadas por punto y coma (;). Cada declaración incluye una **propiedad** CSS y un **valor** separado por dos puntos (:). Este bloque se encierra entre llaves({}).

```css
selector{
    propiedad: valor;
    propiedad2: valor2;
}
```

Ejemplo:

```css
p {
	color: blue;
    font-size: 16px;
}
```

Este ejemplo selecciona todos los elementos <p> (párrafos) y les aplica un color de texto azul y un tamaño de fuente de 16 px.

En CSS, los valores pueden ser palabras especiales o números. A veces, los números llevan unidades de medida. Los valores principales son colores y tamaños.

**Cómo funcionan las reglas CSS en conjunto **

En una hoja de estilo CSS, las reglas se aplican a la página HTML **en el orden en que aparecen**. Cuando existen **reglas contradictorias** para un mismo elemento, el navegador prioriza la **ultima regla leída,** a menos que se especifique lo contrario con la propiedad !important o existan diferencias de especialidad entre los selectores.

El uso de reglas CSS permite a los diseñadores y desarrolladores web separar el contenido (HTML) de la presentación (CSS), lo que facilita la mantenibilidad y reutilización del código, además de permitir una mayor flexibilidad y control sobre el diseño de la página web.



#### **La cascada y la herencia**

La **cascada** y la **herencia** son principios fundamentales que determinan cómo se aplican los estilos a los elementos HTML. Resuelven conflictos cuando múltiples reglas podrían aplicarse al mismo elemento y establecen cómo se transmiten los estilos de un elemento a sus descendientes. 

**Cómo funciona la cascada**

La "cascada" en CSS describe el proceso utilizado para determinar qué reglas de estilo se aplicarán a un mismo elemento cuando múltiples reglas tienen selectores que coinciden con el mismo elemento. La cascada sigue estas reglas básicas para resolver conflictos:

- **Importancia:** Las reglas marcadas con !important tienen la máxima prioridad. Sin embargo, el uso excesivo de !important se considera una mala práctica por que rompe el flujo natural de la cascada.
- **Especificidad:** Cuando dos reglas tienen diferentes selectores, pero podrían aplicarse al mismo elemento, la regla con el selector de mayor especificidad gana. La especificidad se calcula en función de la cantidad y el tipo de componentes del selector (por ejemplo, los ID son más específicos que las clases, y las clases son más específicas que los selectores de tipo).
- **Orden de origen:** Si dos reglas tienen la misma importancia y especificidad, la última regla en el código CSS será la que se aplique.

**Ejemplo de cascada**

```css
/* Regla 1: Selector de tipo que se aplica a todos los elementos <p>*/
p{
    color: red;/* Estable el color de texto de todos los párrafos a rojo */
}
/* Regla 2: Selector de clase que aplica a elementos <p> con la clase 'destacado'*/
.destacado{
    color: blue;/* Establece el color de texto de los párrafos destacados a azul */
}
/* Regla 3: Selector de ID que aplica al elemento <p> con el ID 'especial'*/
#especial{
    color: green; /* Establece el color de texto del párrafo especial a verde */
}
```

Y nuestro HTML podría verse así:

```html
<p>Este párrafo será rojo</p>
<p class="destacado">Este párrafo será azul debido a su clase.</p>
<p id="especial">Este párrafo será verde debido a su ID.</p>
<p class="destacado" id="especial">Este párrafo será verde debido a su ID.</p>
```

Cómo funciona la cascada en este ejemplo: 

- Todos los párrafos recibirán inicialmente el estilo de la Regla 1 (color: red;), ya que esta regla selecciona a todos los elementos <p>.
- El segundo párrafo, al tener la clase destacado, también es seleccionado por la Regla 2. Aquí entra en juego la cascada: aunque la Regla 1 también se aplica, la Regla 2 tiene mayor especificidad debido a que se utiliza un selector de clase, por lo tanto, este párrafo será azul. 
- El tercer párrafo, al tener el ID especial, es seleccionado por la Regla 3. Los ID tienen aún más especificidad que las clases, así que este párrafo será verde, ignorando el color rojo aplicado por la Regla 1. 
- El cuarto párrafo es interesante porque cumple con las condiciones tanto de la Regla 2 como de la Regla 3. En este caso, aunque ambas reglas se aplican, la Regla 3 prevalece porque un ID tiene mayor especificidad que una clase, haciendo que este párrafo también sea verde.



**Principios de herencia**

La herencia en CSS es el mecanismo por el cual **algunos** estilos aplicados a un **elemento padre** se transmiten a sus **elementos hijos.**

No todas las propiedades de CSS se heredan, pero muchas relacionadas con el texto, como color y font-family, lo hacen por defecto.

Hay dos aspectos a considerar con respecto a la herencia:

- **Herencia explícita:** Puedes usar la propiedad **inherit** para aplicar explícitamente a un elemento el valor de una propiedad de su elemento padre.

  ```css
  p{
      color: inherit; /* El color del texto de los parrafos sera el mismo que el de su elemento padre*/
  }
  ```

- **Anulación de la herencia:** Aunque un elemento herede estilos de su elemento padre, cualquier regla aplicada directamente al elemento hijo prevalecerá sobre la herencia.

  ```css
  /* Suponiendo que el elemento padre tenga un color rojo */
  p {
      color: blue; /* Este color azul anulará el color heredado del padre*/
  }
  ```



**Ejemplo de herencia**

Imagina que tenemos un documento HTML que incluye un elemento <div> que a su vez, contiene un elemento <p> (párrafo):

```html
<div class="contenedor">
    Este es un texto dentro del div.
    <p>
        Este es un párrafo dentro del div
    </p>
</div>
```

Y tenemos la siguiente regla CSS aplicada al elemento <div>:

```css
.contedor {
    color: blue; /* Establece el color de texto de los elementos dentro del div a azul */
    font-size: 20px;/* Establece el tamaño de fuente de los elementos dentro del div*/
}
```

Cómo Funciona la Herencia en Este Ejemplo: 

- El elemento <div>  con la clase contenedor se estilizará directamente según las reglas definidas: su texto será azul y tendrá un tamaño de fuente de 20px. 
- El elemento <p> (párrafo) dentro del <div>  no tiene reglas de estilo específicas aplicadas directamente a él en este ejemplo. Sin embargo, debido a la herencia en CSS, heredará algunas de las propiedades de estilo de su elemento padre <div>. En este caso, heredará el color: blue; pero no todas las propiedades se heredan de manera predeterminada, como el margin o el padding.
- Como resultado, el texto dentro del párrafo también aparecerá en azul y con un tamaño de fuente de 20px, a pesar de que no se aplicaron estilos directamente al elemento <p>. Esto demuestra cómo la herencia puede simplificar la aplicación de estilos comunes a varios elementos, manteniendo la coherencia en todo el diseño.



**Excepciones y control de la herencia**

No todas las propiedades CSS se heredan de manera predeterminada. Propiedades como margin, padding, border y background no se heredan, lo que significa que los elementos hijos no adoptarán estos estilos de sus padres a menos que se especifique explícitamente.

Se puede controlar la herencia utilizando las palabras clave:

- **inherit:** fuerza a un elemento a heredar una propiedad específica de su elemento padre.
- **initial:** establece la propiedad a su valor predeterminado, ignorando la herencia.
- **unset:** restablece la propiedad a su valor heredado si se hereda por defecto, o al valor inicial si no.



**Conflictos y solución de problemas en CSS**

A medida que las hojas de estilo se vuelven más complejas y se aplican a documentos HTML igualmente complejos, es inevitable encontarse con **conflictos entre reglas CSS** que compiten por aplicar estilos a los mismos elementos. 

Entender cómo resolver estos conflictos y cómo utilizar las herramientas de desarrollo disponibles en los navegadores modernos es esencial para cualquier desarrollador web.



**Especificidad y conflictos**

La especificidad es un **sistema de puntuación** que los navegadores utilizan para determinar que **reglas CSS tienen prioridad** sobre otras cuando más de una regla podría aplicarse a un elemento. 

La especificidad se calcula en función de los diferentes tipos de selectores utilizados en una regla CSS y su planteamiento jerárquico es el siguiente:

- Los selectores de ID tienen la especificidad más alta.
- Los selectores de clase, atributos y pseudo-clases vienen después.
- Los selectores de tipo y pseudo-elementos tienen la especificidad más baja.
- La declaración !important (aunque se recomienta evitar su uso cuando sea posible) sobrepasa otros niveles de especificidad.

Cuando dos reglas en conflicto tienen la misma especificidad, la regla que aparece la última en el CSS es la que se aplica.

Ejemplo:

```css
#id{
    color: red; /* Especificidad alta*/
}
.clase{
    color: blue; /*Especificidad media, pero no se aplica debido a la regla del ID*/
}
p{
    color: green;/* Especificidad baja, superada por las reglas de clase e ID */
}
```

En este ejemplo, si un elemento <p> con la clase .clase y el ID #id está presente, el color del texto aplicado sería rojo debido a la especificidad más alta del selector de ID. 

### [¿Cómo se calcula la Especificidad?](https://lenguajecss.com/cascada-css/especificidad/especificidad-css/#cómo-se-calcula-la-especificidad)

El navegador tiene un sistema llamado **Especificidad CSS**, donde en situaciones de conflicto como la anterior, calculará que **selector es más específico**, siguiendo unas ciertas normas, y obtienendo como resultado un valor numérico.

Dicho valor numérico se suele representar con 3 cifras, separadas por comas: `A,B,C`:

| Valor         | Descripción                                                  |
| ------------- | ------------------------------------------------------------ |
| **Valor `A`** | Número de veces que aparece un `#id` en el selector.         |
| **Valor `B`** | Número de veces que aparece una `.clase`, `[atributo]` o `:pseudoclase` en el selector. |
| **Valor `C`** | Número de veces que aparece un `elemento` o un `::pseudoelemento` en el selector. |

Teniendo en cuenta esto, veamos algunos ejemplos con diferentes selectores CSS y calculemos su cifra de **especificidad CSS**. Recuerda que cuanto más alta sea, más específico es el selector.

Ejemplo:

```css
div { ... }                   /* 0,0,1 (1 elemento HTML) */
div div { ... }               /* 0,0,2 (2 elementos HTML) */
#pagina div { ... }           /* 1,0,1 (1 id y 1 elemento HTML) */
#pagina div:hover { ... }     /* 1,1,1 (1 id, 1 pseudoclase y 1 elemento HTML) */
#pagina div:hover a { ... }   /* 1,1,2 (1 id, 1 pseudoclase y 2 elementos HTML) */
#pagina .sel:hover>a { ... }  /* 1,2,1 (1 id, 1 clase, 1 pseudoclase y 1 elemento HTML) */
```

Estos cálculos no hace falta hacerlos a mano o mentalmente, existen **calculadoras de especificidad** que pueden darte el valor por si tienes curiosidad por saber cuál tiene mayor o menor especificidad. De hecho, las **DevTools** del navegador, también suelen mostrar la especificidad si mueves el ratón sobre el selector.

Formas alternativas de ver la especificidad de un selector CSS:

- En **Chrome Dev Tools**, moviendo el ratón sobre el selector en **Styles**.
- En la [Calculadora de especificidad CSS de Keegan](https://specificity.keegan.st/)
- En la [Calculadora de especificidad CSS de Polypane](https://polypane.app/css-specificity-calculator/)

Existen **excepciones** en la forma de calcular la especificidad CSS, como por ejemplo en los [combinadores :is() o :where()](https://lenguajecss.com/cascada-css/especificidad/is-where/),

| Excepción               | Cómo se modifica la especificidad de su interior             |
| ----------------------- | ------------------------------------------------------------ |
| 1️⃣ Combinador `:is()`    | Se suma la especificidad del selector más específico de la lista. |
| 2️⃣ Combinador `:not()`   | Usa la misma especificidad del selector que contiene.        |
| 3️⃣ Combinador `:has()`   | Usa la especificidad base (`10`) más la del selector que contiene. |
| 4️⃣ Combinador `:where()` | No importa el selector interno. Su especificidad siempre es `0`. |

Veamos un fragmento de código para afianzar los conceptos de estos combinadores CSS `:is()`, `:not()`, `:has()` y `:where()` con los siguientes ejemplos:

```css
.container { }                       /* Especificidad: 0,1,0 */
.container .item { }                 /* Especificidad: 0,2,0 */

.container :is(.item, #unique) { }   /* 1,1,0 (#unique es el más específico) */
.container :not(.item) { }           /* 0,2,0 */
.container:has(.item) { }            /* 0,2,0 */
.container :where(.item) { }         /* 0,1,0 (solo cuenta el de fuera) */
```







##### Conflictos de nombres CSS

Hay un conflicto o una colisión de nombres CSS cuando tenemos varios fragmentos de código CSS que dan diferentes estilos a un mismo elemento. El navegador debe buscar una forma de resolver ese conflicto, ya que no puede aplicar dos colores diferentes a un mismo elemento, por ejemplo.

Por ejemplo:

#### [Caso 1: Desde diferentes fuentes](https://lenguajecss.com/cascada-css/estructura/conflictos-css/#caso-1-desde-diferentes-fuentes)

html

```css
<html>
<head>
  <link rel="stylesheet" href="index.css">
  <!-- contiene un .text { background-color: blue; } -->
  <style>
    .text { background-color: red }
  </style>
</head>
<body>
  <div class="text" style="background-color: green">
    ¡Desde Manz.dev puedes ver mis streams!
  </div>
</body>
</html>
```

En el siguiente ejemplo se aplican diferentes estilos CSS al mismo elemento:

- 1️⃣ Aplicamos **color azul** desde el `<link>`
- 2️⃣ Aplicamos **color rojo** desde el `<style>`
- 3️⃣ Aplicamos **color verde** desde la propia etiqueta HTML mediante `style`

![image-20250304134109587](/home/soa/.config/Typora/typora-user-images/image-20250304134109587.png)

En este caso, tendría **prioridad** el estilo definido en la propia etiqueta HTML (*estilo en línea*), ya que el orden de prioridad es el siguiente:

- 1️⃣🟥 El atributo `style=""` del HTML (*mayor prioridad*)
- 2️⃣🟧 Bloque de estilos `<style>` en el HTML (*prioridad intermedia*)
- 3️⃣🟨 Archivo .css externo vía `<link rel="stylesheet">` (*menor prioridad*)

Lo recomendable y habitual suele ser escribir los estilos en una hoja de estilos externa (*más fácil de mantener y organizar*). De esta forma, esa hoja externa tendría la menor prioridad. Si en algún caso necesitamos añadir CSS con mayor prioridad, lo haríamos en un bloque `<style>` o en un [estilo en línea](https://lenguajecss.com/cascada-css/estructura/inline-styles/).



#### [Caso 2: De fuentes del mismo tipo](https://lenguajecss.com/cascada-css/estructura/conflictos-css/#caso-2-de-fuentes-del-mismo-tipo)

otro ejemplo. Supongamos ahora que nos encontramos en la misma fuente (*por ejemplo, una hoja de estilos externa*). Vamos a aplicar estilos a **exactamente el mismo selector**, en este caso, al elemento `.text`.

La duda sería: ¿Cuál de las dos propiedades `background-color` prevalece? Se refieren al mismo elemento por **exactamente el mismo selector** y están en la misma fuente de datos:

```css
.text {
  padding: 8px;
  background-color: red;
}

.text {
  background-color: grey;
  color: white;
}
```

```html
<div class="text">Texto del elemento</div>
```

La respuesta es muy fácil: **Prevalece siempre la última regla definida** (orden). Como el selector es **exactamente el mismo**, entonces el navegador mezcla los estilos de ambos bloques y se fusionan. El resultado final interpretado por el navegador (*valor computado*) sería el siguiente:

```css
.text {
  padding: 8px;             /* No hay conflicto, se añade */
  background-color: grey;   /* Hay conflicto: se sobreescribe la última */
  color: white;             /* No hay conflicto, se añade */
}
```

![image-20250304180540065](/home/soa/.config/Typora/typora-user-images/image-20250304180540065.png)



#### [Caso 3: Diferente selector](https://lenguajecss.com/cascada-css/estructura/conflictos-css/#caso-3-diferente-selector)

Puede ocurrir que en algunos casos no esté claro cuál es el estilo que debería sobreescribir a los anteriores, porque tenemos **selectores diferentes** que se refieren **al mismo elemento**.

Por ejemplo, en este caso tenemos un elemento HTML con una `clase` y con un `id`. Si a este código HTML le aplicamos el código CSS del bloque `<style>`, aunque los **selectores sean diferentes**, nos estamos refiriendo al mismo elemento, por lo que no resulta tan intuitivo como vimos al principio de este artículo:

```html
<div class="text" id="element">¡Hello from Manz.dev!</div>

<style>
    #element {
      background: blue;
    }

    .text {
      background: red;
    }
</style>
```

Aquí es cuando entra en juego el concepto de **Especificidad CSS**. Este concepto se encarga de eliminar la ambigüedad determinando **cuál es el selector más específico** que debería «vencer».

Para entenderlo mejor, veamos otro fragmento de CSS aplicado a la etiqueta HTML `<div>` anterior:

```html
<div id="element" class="text">Texto del elemento</div>

<style>
div { background-color: red; }
#element { background-color: steelblue; }
.text { background-color: green; }
</style>
```

![image-20250304181154127](/home/soa/.config/Typora/typora-user-images/image-20250304181154127.png)

Si nos fijamos en el HTML, nuestro único `<div>` encaja con los tres selectores del CSS:

- 1️⃣ Es un `<div>` por lo que se le aplica el color rojo.
- 2️⃣ Tiene el `id` con valor `element`, por lo que se le aplica el color azul.
- 3️⃣ Tiene la `clase` con valor `text`, por lo que se le aplica el color verde.

El navegador analiza cuál es el selector más específico, para descubrir cuál es el vencedor:

- El selector `div` tiene especificidad `0,0,1` (*es un elemento*)
- El selector `#element` tiene especificidad `1,0,0` (*es un `id`*)
- El selector `.text` tiene especificidad `0,1,0` (*es una `class`*)

De estos tres selectores, el vencedor es el segundo (*número más alto*): el selector `#element`, ya que los elementos con `id` no se pueden repetir en un mismo documento, y son más específicos.



#### [Importancia: Excepción `!important`](https://lenguajecss.com/cascada-css/estructura/conflictos-css/#importancia-excepción-important)

Existe una forma de saltarse todo lo anterior mediante una palabra clave llamada `!important`.

Hay que tener en cuenta que su uso debería estar siempre justificado y no utilizarse a la ligera. Con `!important` podemos dar **máxima prioridad** a un valor sobre una propiedad, dando igual la especificidad que tenga, el orden en el que esté o la fuente donde haya sido declarada.

### 3.2 Selectores, propiedades y valores.

Los selectores y propiedades son herramientas esenciales que determinan cómo se presenta el contenido en la web.

#### Los selectores

Permiten dirigir los estilos a elementos específicos en la página, desde elementos individuales identificados por ID hasta grupos de elementos clasificados por clases o tipos.

Son el primer paso para aplicar estilos a tu página web. Actúan como el **puente entre el documento HTML y las reglas de estilo CSS,** determinando exactamente qué elementos del documento serán afectados por las declaraciones de estilo.

Un selector en CSS **identifica los elementos HTML** a los que se aplicará un conjunto de reglas de estilo. Los selectores pueden apuntar a elementos por su tipo de etiqueta, clase, ID, atributos y más, permitiendo una gran precisión y flexibilidad en la aplicación de estilos. También pueden utilizarse desde ciertos lugares de Javascript. 







**Tipos de selectores:**

- **Selectores de tipo:** o selectores de etiqueta, estos aplican estilos a todos los elementos de un tipo específico.

  ```css
  p{
  	color: blue;
  }
  ```

  El selector p seleccionará todos los elementos <p> en el documento y aplicará a ellos las reglas de estilo definidas: color azul.

- **Selectores de clase:** Permiten estilizar un grupo de elementos que comparten la misma clase HTML. Se denotan por un punto . seguido del nombre de la clase.

  ```css
  .destacado {
  	font-weight: bold;
  }
  ```

  El selector .destacado aplicará estilos a todos los elementos que tengan class="destacado".

- **Selectores de ID:** Se utilizan para estilizar un elemento único en el documento, identificado por su atrinbuto id. Se denotan con un signo de hash # seguido del valor del ID.

  ```css
  #principal {
  	background-color: yellow;
  }
  ```

  El selector #principal aplicará estilos al elemento que tenga id="principal".

- **Selector universal:** se denota por *, selecciona todos los elementos en el documento. Aunque es útil en ciertos contextos, su uso excesivo puede llevar a problemas de rendimiento y sobreescritura no deseada de estilos.

  ```css
  * {
  	box-sizing: border-box;
  }
  ```

  El selector universal aplicará esta propiedad a todo el HTML al que esté aplicando estilo.



Cada tipo de selector tiene su propio nivel de especificidad y uso ideal, lo que permite a los diseñadores y desarrolladores web aplicar estilos de manera precisa y eficiente a los elementos deseados en la página.



#### Las propiedades

Definen qué aspectos del diseño se modificarán, como el color, la tipografía, tamaño, es espaciado y la disposición.

Las propiedades y valores en CSS son los componentes que realmente definen el estilo de los elementos en una página web. Mientras que los selectores determinan qué elementos se estilizan, las propiedades y valores especifican cómo se estilizan estos elementos.

Cada propiedad tiene un conjunto de valores posibles que determinan cómo se mostrará el elemento en la página.

Las propiedades se escriben dentro de un bloque de declaración en una regla CSS, siguiendo el patrón "propiedad: valor;".

### 3.3 Modelo de caja y posicionamiento.

Imagina que una página web es como un conjunto de **cajas.** Cada elemento en la página, desde los mas grandes (como un contendor <div>) hasta los más pequeños (como un texto dentro de un <span>), se representa como una **caja rectangular** en este modelo.

Por ejemplo, imagina un simple artículo de blog compuesto por un título (<h1>), un subtítulo (<h2>), y varios párrafos de texto (<p>). En el modelo de caja, el título, el subtítulo y cada párrafo se consideran cajas individuales, cada una con su propio contenido, relleno, borde y margen, que pueden ser estilizados y posicionados de manera independiente en la página. 

Estas cajas interactúan entre sí y con el contenedor principal de la página. Las personas diseñadoras web utilizan esta estructura para tener un **control preciso** sobre cómo se muestran los elementos en la página. Así pueden crear **diseños complejos y responsivos** de manera más fácil.

**Elementos de una caja**

Cada "caja" en el modelo de caja de CSS consta de cuatro componentes principales: 

- **Contenido (Content):** El área donde se muestra el contenido real del elemento, como texto o imágenes. Las propiedades **width** y **height** controlan el tamaño de esta área. 
- **Relleno (Padding)**: El espacio entre el contenido del elemento y su borde. Aumenta el área total del elemento, pero no el tamaño del contenido en sí. Se puede controlar individualmente con **padding-top, padding-right, padding-bottom, y padding-left**. 
- **Borde (Border)**: Encierra el contenido y el relleno. Puede ser visible o invisible y se controla con propiedades como **border-style, borderwidth, y border-color.** 
- **Margen (Margin)**: El espacio alrededor de la caja, separándola de otros elementos. Al igual que el relleno, puede controlarse individualmente para cada lado del elemento.

La suma de estas áreas (contenido, relleno, borde y margen) determina el tamaño total y la posición de un elemento en la página.

##### Margin collapse (colápso de márgenes):

CSS tiene varios mecanismos de maquetación, inline y block, en ellos ocurre la siguiente situación. Imaginemos que tenemos dos elementos adyacientes, y cada uno de ellos, tiene un margen de 20px definido.

Quizás, lo más intuitivo es pensar que el elemento de la izquierda tiene 20px de margen derecho, mientras que el elemento de la derecha tiene 20px de margen izquierdo, que al sumarse, predecimos que resultará un tamaño de 40px. Sin embargo la realizad es distinta:

![image-20250306142957947](/home/soa/.config/Typora/typora-user-images/image-20250306142957947.png)

El **margin collapse** es una situación que ocurre en estos modelos de maquetación básica, donde los márgenes de dos elementos se colapsan, ocupando el mismo tamaño a la vez. En el ejemplo anterior, ocupa sólo esos 20px, en lugar de los 40px de la suma del margen de cada caja.

En los modelos de maquetación avanzados como **flex** o **grid** no se aplica el margin collapse, ya que es algo que sólo afecta a modelos básicos como inline o block.







**Ejemplo**:

Imagina que estás construyendo una página web como si fuera un juego de bloques. Como un Lego. Cada parte de tu página web (como un párrafo de texto, una imagen o un botón) es un bloque. En el mundo del diseño web, llamamos a estos bloques **cajas**. 

Piensa en una página web como si fuera un montón de cajas, unas dentro de otras. Cada cosa en la página ya sea grande como un contenedor <div>, o pequeña como un trozo de texto <span> , es una caja. Los diseñadores web pueden cambiar el estilo y la posición de cada caja para crear páginas web atractivas y fáciles de usar. Cada caja tiene su propio contenido, que es el texto que lees. Además, tiene relleno, que es el espacio alrededor del texto. Asimismo, tiene borde, es decir, la línea alrededor del relleno, y tiene margen, que es el espacio alrededor de la caja.

Imagina que estás creando una tarjeta de perfil para un sitio web. Cada tarjeta es un elemento que contiene una foto, un nombre y una breve descripción.

**Elementos de una caja:** 

- **Contenido**: Es donde pones cosas como texto o imágenes. Aquí sería la foto, el nombre y la descripción dentro de la tarjeta. Puedes cambiar el tamaño de esta área con las propiedades **width** (ancho) y **height** (alto). 

- **Relleno** (padding): Se trata del espacio entre los **bordes** de la foto, el nombre y la descripción y los límites de la tarjeta.  Añadir padding alrededor del contenido garantiza que no esté pegado directamente a los bordes de la tarjeta, haciendo que el diseño sea más atractivo visualmente. 

  Aumenta el tamaño total de la caja, pero no el tamaño del contenido. Puedes controlar el relleno en cada lado de la caja con las opciones de **padding-top, padding-right, padding-bottom y padding-left**.

  La propiedad `padding` se puede especificar utilizando uno, dos, tres o cuatro valores. Cada valor es una longitud o un porcentaje. Los valores negativos no son válidos.

  - Cuando se especifica **un** valor, se aplica el mismo relleno a **los cuatro lados**.
  - Cuando se especifican **dos** valores, el primer relleno se aplica a la parte de **arriba y abajo**, el segundo a la **izquierda y la derecha**.
  - Cuando se especifican **tres** valores, el primer relleno se aplica a la parte de **arriba**, el segundo a la **derecha e izquierda**, el tercero a la parte de **abajo**.
  - Cuando se especifican **cuatro** valores, los rellenos se aplican a la parte de **arriba**, **derecha**, **abajo** e **izquierda** en ese orden (en el sentido de las agujas del reloj).

  Ejemplo:

  ```css
  /* Aplica a los cuatro lados */
  padding: 1em;
  
  /* arriba y abajo | izquierda y derecha */
  padding: 5% 10%;
  
  /* arriba | izquierda y derecha | abajo */
  padding: 1em 2em 2em;
  
  /* arriba | derecha | abajo | izquierda */
  padding: 5px 1em 0 2em;
  
  /* Valores globales */
  padding: inherit;
  padding: initial;
  padding: revert;
  padding: revert-layer;
  padding: unset;
  ```

  

- **Borde**: Es la línea que rodea el contenido y el relleno. Si decides que cada tarjeta debería tener un borde para distinguirla claramente del fondo de la página o de otras tarjetas, este sería el borde alrededor del contenido y el relleno. Puede ser visible o invisible y puedes cambiar su estilo, ancho y color con **border-style, border-width y border-color** para definir su apariencia.

- **Margen** (margin): Ayuda a separar una caja de otras cajas. Al igual que el relleno, puedes controlar el margen en cada lado de la caja. El espacio alrededor de la tarjeta que la separa de otras tarjetas o elementos en la página. La suma de estas áreas determina el tamaño total y la posición de un elemento en la página. Usar margin asegura que las tarjetas no estén demasiado apretadas unas contra otras, mejorando la legilibilidad y la estética general del diseño.



#### Box-sizing

Una vez que entiendes las partes principales de una caja (contenido, relleno, borde y margen),  entra en juego la regla CSS de **box-sizing**.: una **regla CSS** que nos ayuda a entender cómo se calcula el tamaño total de la caja.

Por defecto, el ancho y la altura de una caja se aplican solo al área de contenido, lo que significa que el relleno y el borde se añaden al tamaño final del elemento. 

Por defecto, CSS utiliza el valor **content-box** para la propiedad box-sizing, lo que implica que cuando especificas el width(ancho) y el height (altura) de un elemento, estas medidas se aplican únicamente al área del contenido de la caja. Esto significa que el relleno (padding) y el borde (border) que añades a este elemento no se incluyen dentro de las dimensiones especificiadas para el ancho y la altura; en cambio, se suman a estas automáticamente, aumentando el tamaño total del elemento en la página.

Por ejemplo, si tienes un elemento con un width de 100 px y un height de 100 px, y le añades un padding de 10px por todos los lados y un border de 2px, el tamaño total del elemento no será de 100x100px. En cambio, debido al relleno y al borde adicionales, el tamaño final será de 124px de ancho (100px de contenido + 10px de relleno de la izquierda + 10px de relleno a la derecha + 2px de borde a la izquierda + 2px de borde a la derecha) y 124px de alto, siguiendo la misma lógica.

Este comportamiento puede ser no intuitivo, especialmente para quienes están aprendiendo CSS o para aquellos casos en los que se requiere un control preciso del layout, ya que el tamaño final de los elementos puede terminar siendo mayor de lo esperado, afectando la disposición de estos en la página. Por ello, es esencial comprender cómo el valor predeterminado de box-sizing influye en el cálculo de las dimensiones de los elementos y cómo puedes modificar este comportamiento utilizando la propiedad box-sizing para ajustarse mejor a tus necesidades de diseño.

La propiedad **box-sizing** utiliza dos valores principales para controlar como se calculan las dimensiones de un elemento. Hay dos valores principales:

- **content-box** (valor por defecto): El ancho y la altura del elemento se aplican solo al contenido, y el relleno y el borde se añaden al total del elemento.

- **border-box**: el ancho y la altura del elemento incluyen el contenido, el relleno y el borde, lo que facilita el diseño al hacer que los elementos se comporten de manera más predecible.

  ```css
  .box {
  	box-sizing: border-box;
  }
  ```

  



#### Posicionamiento en CSS

El posicionamiento en CSS es una herramienta poderosa que afecta **cómo y dónde se muestra los elementos en la página web.**

Una de las partes más complejas y fundamental de CSS es la creación de layouts, colocación y distribución de los elementos a lo largo de una página.

layout: algoritmo que determina la posición y tamaño de las cajas basado en la forma en la que interacúan con sus (elementos) hermanos y padres.

Para establecer estos mecanismos básicos de layout se utiliza una propiedad llamada **display**, que aunque no la estemos utilizando explícitamente, está actuando por omisión.

##### flex (flexbox)

Inicialmente se utilizaba el posicionamiento (static, relative, absolute), los elementos en línea o en bloque (y derivados) o la propiedad **float** para realizar maquetaciones.

Es un sistema de **elementos flexibles** más potente, limpia y personalizable que lo anterior, en la que los elementos HTML se adaptan y colocan automáticamente. 

**Flex** está especialmente diseñado para crear, mediante CSS, estructuras de **una sóla dimensión**

![image-20250307112258085](/home/soa/.config/Typora/typora-user-images/image-20250307112258085.png)

- **Contenedor:** es el elemento padre que tendrá en su interior cada uno de los ítems flexibles. Al contrario que muchas otras estructuras CSS, por norma general, en Flex establecemos las propiedades al elemento padre.
  - Eje principal: los contenedores flexibles tendrán una orientación principal específica. Por defecto el contenedor flex es en **horizontal** (en fila).
  - **Eje secundario:** de la misma forma, los contenedores flexibles tendrán una orientación secundaria, perpendicular a la principal. Si la principal es en horizontal, la secundaria será en **vertical ** (y viceversa).
- **Ítem:** cada uno de los **hijos** que tendrá el contenedor en su interior.

Ejemplo:

```html
<div class="container"> <!-- Flex container -->
  <div class="item item-1">1</div> <!-- Flex items -->
  <div class="item item-2">2</div>
  <div class="item item-3">3</div>
</div>
```

Para activar el modo **flex**, utilizaremos sobre el elemento contenedor la propiedad **display** y especificaremos el valor **flex o inline-flex** (dependiendo de como queramos que se comporte el contenedor).

-  **inline-flex**: establece un contenedor en línea, similar a **inline-block** (ocupa solo el contenido).
- **flex:** establece un contenedor en bloque, similar a **block** (ocupa todo el ancho del padre).

Por defecto, y sólo con esto, observaremos que los elementos se disponen todos sobre una misma línea. Esto ocurre porque estamos utilizando el modo **flex**

###### Contenedor flex multilínea

En general, **flex**, se suele utilizar para estructuras de una sola dimensión, es decir contenedores que sólo van en una dirección. Sin embargo, existe una propiedad **flex-wrap** con la que podemos especificar un comportamiento especial del contenedor.

Por defecto, si un elemento no cabe dentro de nuestro contenedor flex, los elementos se harán más pequeños (son flexibles) para ajustarlos al contenedor.

Este es el comportamiento por defecto de un contenedor **flex**. Sin embargo, con la propiedad **flex-wrap** podemos cambiar este comportamiento y permitir que nuestro contenedor flex, se desborde, conviertiéndose en un **contenedor flex multilínea.**

- **flex-wrap:** evita o permite el desbordamiento (multilínea). Posibles valores:

  - **nowrap:** los ítems se ajustan para ocupar el tamaño del contenedor. (No permite desbordamiento en múltiples líneas).
  - **wrap:** Establece los ítems en modo multilínea (permite que se desborde el contenedor).
  - **wrap-reverse:** Establece los ítems en modo multilínea, pero en dirección inversa.

  ```css
  .container {
    display: flex;
    flex-wrap: wrap;
    background: steelblue;
    width: 200px;
  }
  
  .item {
    background: grey;
    width: 50%;
  }
  ```

  En el caso de especificar **nowrap** (u omitir la propiedad flex-wrap) en el contenedor, los 3 ítems se mostrarían en una misma línea del contenedor (que es el comportamiento por defecto). En ese caso, si tenemos 3 ítems, cada uno debría tener aproximadamente 66px de ancho. Un tamaño de 100 px por ítem, sumaría un total de 300px, que no cabrían en el contenedor de 200px, por lo que **flex** reajusta los ítems flexibles para que quepan todos en la misma línea, manteniendo las mismas proporciones.

  Sin embargo, si especificamos **wrap** en la propiedad **flex-wrap**, lo que permitimos es que el contenedor se pueda desbordar, pasando a ser un contenedor **multilínea**, que mostraría los ítems que quepan en la primera línea y el resto en las líneas siguientes. El valor **wrap-reverse** haría exactamente lo mismo, pero con el orden inverso.

Existe una propiedad de atajo (short-hand) llamada **flex-flow**, con la que podemos resumir los valores de las propiedades **flex-direction** y **flex-wrap**, especificándolas en una sola propiedad y ahorrándonos utilizar las propiedades concretas:

```css
}
```

Existen dos propiedades de flexbox que han surgido recientemente: **row-gap** y **column-gap**. Dichas propiedades, permiten establecer el tamaño de un "hueco" entre ítems desde el elemento padre contenedor, y que eliminan la necesidad de estar utilizando **padding** o **margin** en los elementos hijos, con las complicaciones que ello suele conllevar:

- **row-gap:** espacio entre filas (sólo funciona con flex-direction: column)
- **column-gap:** espacio entre columnas (sólo funciona con flex-direction: row)

Ten en cuenta que, como **flex** es un sistema para diseños de una sola dimensión, sólo una de las dos propiedades tendrá efecto. Si la propiedad **flex-direction** está establecida en **column**, podrás utilizar **row-gap**, y en el caso de que la propiedad **flex-direction** se encuentre en **row**, podrás utilizar el **column- gap**.

Eso sí, es posible usar ambas si tenemos la propiedad **flex-wrap** definida a **wrap** y, por lo tanto, disponemos de multicolumnas flexbox, ya que en ese caso si podemos separar elementos por filas y por columnas.

**Los huecos sólo se aplican entre los elementos, y no entre un elemento hijo y su hijo contenedor padre.**

##### Tipos de posicionamiento

CSS ofrece varios tipos de posicionamiento, cada uno con su propósito y comportamiento específicos:

- **static**

  Es el valor predeterminado. Los elementos se colocan en el flujo normal del documento, uno tras otro, según aparecen en el HTML.

  - Uso típico: No necesitas definirlo porque todos los elementos HTML lo tienen por defecto. Es ideal cuando quieres que los elementos sigan el flujo normal de la página, uno tras otro.
  - Ejemplo: un párrafo de texto seguido por una imagen, tal como aparecen en tu código HTML.

- **relative**

  Posiciona un elemento en relación con su posición original en el flujo del documento. Puedes mover un elemento con top, right, bottom y left, pero el espacio original que ocupaba el elemento pertenece.

  - Uso típico: Perfecto para hacer pequeños ajustes de posición a un elemento sin afectar el layout general.
  - Ejemplo: Mover ligeramente un botón hacia abajo para alinearlo mejor con otros elementos de su alrededor.

- **absolute**

  Extrae el elemento del flujo normal del documento y lo posiciona en relación con su contenedor posicionado más cercano. Si no hay tal contendor, se posiciona en relación con el elemento <html>.

  - Uso típico: Ideal para posicionar un elemento exactamente donde lo deseas en la página, como una ventana modal o un tooltip.
  - Ejemplo: un mensaje emergente (tooltip) que aparece sobre un elemento para ofrecer información adicional. 

- **fixed**

  Similar a absolute, pero el elemento se posiciona en relación con la ventana del navegador, permaneciendo fijo durante el deplazamiento.

  - Uso típico: usado comúnmente para elementos de navegación o botones que necesitas que se queden visibles todo el tiempo, sin importar el desplazamiento de la página.
  - Ejemplo: Una barra de navgación que permanece en la parte superior de la pantalla incluso cuando desplazas hacia abajo.

- **sticky**

  Un híbrido entre relative y fixed. El elemento se posiciona como relative hasta que alcanza un punto de desplazamiento específico, momento en el cual se convierte en fixed.

  - Uso típico: Útil para elementos que quieres que se comporten de manera normal hasta que llegues a un punto específico de desplazamiento, momento en el cual se fijan en su lugar.
  - Ejemplo: Un encabezado de sección en una lista o tabla que se queda en la parte superior mientras desplazas por esa sección.



#### Layouts con modelo de caja y posicionamiento

Un layout en diseño web es la **forma en que se organizan las cosas en una página web.** Esto incluye la disposión de componentes como los títulos, el pie de página, las columnas de texto, las barras laterales, las imágenes, los botones y los enlaces.

El propósito de un buen layout es mostrar la información de manera que sea **fácil de entender** y accesible.

A la hora de crear ese layout:

- el modelo de caja define cómo se estructura y dimensionan los elementos en una página, considerando margen, borde, relleno y contenido. Es, por tanto, ideal para ajustar el espacio interior y exterior de los elementos, controlar su tamaño y gestionar cómo interactúan entre sí el flujo normal del documento.
- El posicionamiento, por otro lado, determina la ubicación exacta de estas cajas en el layout de la página. Es ideal para ajustar el espacio interior y exterior de los elementos, controlar su tamaño y gestionar como interactúan entre si en el flujo normal del documento.



#### Diseño responsivo

El diseño responsive o responsive desingn, es una forma de hacer páginas web que se ven y funcionan bien en **todo tipo de dispositivos y tamaños de pantalla,** desde teléfonos hasta ordenadores. La idea es que el diseño de la página web **cambie según el dispositivo** del usuario, adaptándose a cosas como el tamaño de la pantalla, el sistema operativo y si el dispositivo está en posición vertical u horizontal.

El modelo de caja y el posicionamiento en CSS son fundamentales para implementar un diseño responsivo, ya que determinan cómo se organizan y visualizan los elementos en la página.

##### Relación con el modelo de caja y posicionamiento

El modelo de caja y el posicionamiento en CSS son fundamentales para implementar un diseño responsivo, ya que determinan cómo se organizan y visualizan los elementos en la página.

- **Modelo de caja:**

  La forma en que definimos el margen, el borde, el relleno y las dimensiones de los elementos afecta directamente cómo se adaptan a diferentes tamaños de pantalla. Por ejemplo, el uso de unidades relativas (como %, em, rem, vw, vh) en lugar de unidades fijas (como px) permite que los elementos se escalen de manera más fluida. Así un elemento puede ocupar width:100% en un teléfono móvil para aprovechar todo el ancho disponible, mientras que en una pantalla más grande puede limitarse a width: 50% para no extenderse demasiado.

- **Posicionamiento:**

  Las técnicas de posicionamiento permiten controlar la ubicación de los elementos en la página. En un diseño responsivo, podemos cambiar el posicionamiento de los elementos según el tamaño de la pantalla mediante media queries. Por ejemplo una barra lateral position: fixed; en una pantalla de escritorio puede convertirse en position:static; en dispositivos móviles para fluir normalmente dentro del contenido y no ocupar espacio valioso en la pantalla.

Básicamente, se define un layout predeterminado que generalmente está orientado a cierto tamaño de pantalla ( a menudo el tamaño más común o el que consideras principal para tu audiencia) y luego se definen otros layouts aplicables en otros tamaños de pantalla ( por ejemplo, un tablet o un iphone). Son las media queries incluidas en la programación de la página quienes gobiernan la aplicación de esos diferentes layouts.

##### Media queries

Las **media queries** (MQ) son una herramienta poderosa en CSS que permite introducir puntos de quiebre (breakpoints). Estos puntos de quiebre actúan como condiciones que, cuando se cumplen (por ejemplo, cuando la ventana del navegador alcanza un cierto ancho), permiten que se apliquen estilos CSS diferentes. De esta manera, las media queries "gobiernan" que layout o conjunto de estilos se aplica en función de las características del dispositivo, como el tamaño de la pantalla, la resolución, la orientación (vertical u horizontal), ect.

A fin de cuentas, se trata de una especie de condicionales de diseño, en las que si cumple la condición se aplicarán unos estilos, y en caso contrario, se aplicarán otros.

###### La regla @media

Las reglas media queries se indican en el código mediante la regla @media, indicando la condición en cuestión entre paréntesis. La sintaxis sería la siguiente:

| Regla                                  | Descripción                                                  |
| -------------------------------------- | ------------------------------------------------------------ |
| @media (<condicion>)                   | Si se cumple la condición, se aplican los estilos de su interior. |
| @media not( <condicion>)               | Si no se cumple la condición, se aplican los estilos de su interior. |
| @media only (<condicion>)              | Si se cumple la condición y es un navegador moderno, se aplican los estilos de su interior. |
| @media (<condicion>) and (<condicion>) | Si se cumplen ambas condiciones, se aplican los estilos.     |

Si utilizamos la palabra calve **not** antes de los paréntesisi, invertimos la condición.

Ejemplo:

```css
@media (*condición*) {
  .container {
    background: green;
  }
}

@media not (*condición*) {
  .container {
    background: red;
  }
}
```

Si cumple la condición, se aplicará un color verde, en caso contrario aplicará un color rojo. Es similiar al funcionamiento de un if / else en programación.

Al escribir una regla @media podrías estar sobreescribiendo los estilos CSS en otro fragmento posterior. Una buena forma de escribir **MQ** sería escribir las reglas **@ media** siempre al final, como excepciones al código anterior.



El número de bloques de reglas **@media** a utilizar depende del desarrollador web, ya que no hay una obligación o norma de utilizar un número concreto. Se pueden utilizar desde un sólo media query, hasta múltiples de ellos a lo largo del documento CSS.

Ejemplo múltiples condiciones:

```css
@media (*condicion*) and (*condicion){
    .container {
        background: orangered;
    }
}
```

Al igual que **not**, también existe la palabra clave **only**. Suele usarse a modo de **hack**. El comportamiento por defecto ya incluye los dispositivos que encajan con la condición, así que con **only** conseguimos que navegadores antiguos (no la entienden) no procesen la información de esta regla, consiguiendo una forma de dar estilo sólo en navegadores modernos.

En los ejemplos anteriores hemos indicado *condiciones* en el interior de los paréntesis, pero no hemos visto como definir ninguna. Vamos a profundizar en esto.

###### Media Queries (range syntax)

Aunque existen otras formas, la preferida de escribir Media Queries es utilizando la modalidad de **rangos de condiciones** (Media Query Range Syntax), mucho más versátiles que las sintaxis anteriores, y mucho menos tediosas.

Para ello, vamos a escribir las condiciones utilizando operadores de comparación como **<,<=,> o >=**:

```css
.container{
    background: var(--color, grey);
    min-height: 200px;
    padding: 1rem 2rem;
}
@media (width <= 550px) {
  .container {
    --color: indigo;
  }
}

@media (width >= 750px) {
  .container {
    --color: green;
  }
}
```

**Anidamiento en Nesting CSS**

Una de las ventajas del nesting nativo es la posibilidad de **anidamiento** de varios tipos:

- **Anidamiento sobre el padre:**

  En los casos anteriores siempre hemos colocado el selector **&** al principio. Sin embargo, puede darse el caso que queramos anidar de una forma diferente. En el siguiente ejemplo, el selector equivalente sería .item .container, es decir, elementos .item que contengan al padre:

  ```css
  .container {
    width: 800px;
    height: 300px;
    background: grey;
  
    .item & {
      background: green;
      height: 100px;
    }
  }
  ```

   también nos permite hacer anidaciones menos directas pero que pueden ser muy interesantes por su relación con el elemento contenedor. Observa los siguientes ejemplos:

  ```css
  .container {
    width: 800px;
    height: 300px;
    background: grey;
  
    .item & .item & { }     /* Equivalente a -> .item .container .item .container */
    :is(.page, .menu) & {}  /* Equivalente a -> .page .container, .menu .container */
    :not(&) { }             /* Equivalente a -> :not(.container) */
  }
  ```

- **Anidamiento de reglas:**

  Es posible anidar reglas CSS en el interior de bloques de código CSS donde se aplica CSS Nesting. Concretamente, es posible hacerlo si se trata de reglas @media, @supports, @layer, @scope o @container. De esta forma, podemos organizar, por ejemplo código responsive de un fragmento de código de forma muy compacta:

  ```css
  .container {
    height: 200px;
    background: grey;
  
    @media (orientation: landscape) {
      height: 100vh;
    }
  }
  ```

  En este caso, el código del interior de la regla `@media` se le aplica al elemento `.container`, simplificando mucho la sintaxis y haciéndolo más fácil de organizar.

**Reglas:**

- **@supports:** permite establecer fragmentos de código CSS condicionales, aplicando estilos CSS sólo cuando se cumplen ciertas condiciones y restricciones. Esto puede ser muy útil para aplicar unos estilos si el navegador soporta una característica, o aplicar un estilo diferente como fallback si no lo hace.

  | Formato                                           | Descripción                                                  |
  | ------------------------------------------------- | ------------------------------------------------------------ |
  | `@supports (`*condición*`)`                       | Aplica los estilos si se cumple la condición.                |
  | `@supports not (`*condición*`)`                   | Aplica los estilos si no se cumple la condición.             |
  | `@supports (`*condición1*`) and (`*condición2*`)` | Aplica los estilos si se cumplen las diferentes condiciones. |
  | `@supports (`*condición1*`) or (`*condición2*`)`  | Aplica los estilos si se cumple una de las dos condiciones.  |
  | `@supports selector(`*selector*`)`                | Aplica los estilos si soporta el selector indicado.          |
  | `@supports font-tech(`*tech*`)`                   | Aplica los estilos si soporta una tecnología de tipografías. |
  | `@supports font-format(`*format*`)`               | Aplica los estilos si soporta un formato de tipografías.     |

  ```css
  @supports (display: grid) {
    .content {
      display: grid;
      grid-template-columns: 1fr 1fr;
    }
  }
  ```

  En este ejemplo, se definen unos estilos para la clase `.content` que el navegador sólo aplicará en el caso de que tenga soporte para la propiedad `display` con el valor `grid`, es decir, que tenga soporte de [Grid CSS](https://lenguajecss.com/css/maquetacion-y-colocacion/grid-css/). Podemos hacer esto mismo con cualquier otra propiedad CSS, y utilizar la regla `@supports` para crear códigos condicionales.

  Podemos crear también **reglas compuestas** por ejemplo, combinemos una regla de negación con una normal:

  ```css
  @supports not (display: grid) and (display: flex) {
    .content {
      display: flex;
      justify-content: center;
    }
  }
  ```

  En este ejemplo creamos una clase `.content` con contenido estructurado con [flexbox](https://lenguajecss.com/css/maquetacion-y-colocacion/flex/), siempre en el caso de que el navegador no soporte **Grid CSS** pero si soporte **Flex**. Observa que hemos combinado tanto el `not` que afecta sólo a la primera condición, como el `and` que afecta a ambas y exige que se cumplan ambas.

  Si quisieramos crear una doble condición con ambas negadas, deberíamos hacer similar a este ejemplo:

  ```css
  @supports (not (display: flex)) and (not (display: grid)) {
    .box {
      display: inline-block;
    }
  }
  ```

  Esto podría ser interesante, pero recuerda no utilizarlo con propiedades muy antiguas. La regla `@supports` fue implementada en navegadores alrededor del año 2019-2020, por lo que utilizarla para excluir navegadores muy antiguos no funcionará porque tampoco tendrán la regla `@supports` implementada.

  ### [Reglas con fallbacks](https://lenguajecss.com/css/compatibilidad-navegadores/la-regla-supports/#reglas-con-fallbacks)

  En lugar de establecer reglas negadas compuestas, es mejor utilizar un enfoque donde establezcas unos estilos generales, que se sobreescriben si el navegador tiene implementadas nuevas características, como explicaremos a continuación.

  En el siguiente ejemplo verás un primer bloque de código fuera de reglas, que se aplicará en cualquier navegador, moderno o antiguo. Sin embargo, a continuación tenemos dos reglas `@supports` que se ejecutarán en navegadores más actuales:

  ```css
  .content {
    display: inline-block;
  }
  
  @supports (display: grid) {
    .content {
      display: grid;
      grid-template-columns: 1fr;
      justify-content: center;
    }
  }
  
  @supports not (display: grid) and (display: flex) {
    .content {
      display: flex;
      justify-content: center;
    }
  }
  ```

  En el caso de tratarse de un navegador que implemente **Grid CSS**, establecerá los estilos indicados en el primer bloque con la regla `@supports`. Luego, la siguiente regla `@supports` se ejecutará sólo en el caso de que el navegador no soporte **Grid CSS** pero si **Flexbox**.

- **@layer:**

  Con esta regla CSS permite declarar las llamadas **capas de cascada**, concepto muy similar a las capas de un editor gráfico como Photoshop, GIMP o Inkscape: crean diferentes grupos, sólo que en nuestro caso en lugar de colocar imágenes en cada grupo, colocamos código CSS.

  La finalidad de las capas de cascada, es organizar mejor nuestro código, evitar los problemas de especificidad CSS y reducir la necesidad de utilizar !important o reescribir los selectores de CSS:

  ```css
  @layer nombre {
    /* Reglas CSS */
  }
  ```

  Formas para crear capas CSS utilizando la regla @layer, o incluso la regla @import. Su sintaxis es:

  | Formato                                      | Descripción                                                  |
  | -------------------------------------------- | ------------------------------------------------------------ |
  | `@layer` *nombre*                            | Crea una capa CSS con el nombre indicado.                    |
  | `@layer`                                     | Crea una capa CSS anónima (sin nombre).                      |
  | `@layer` *nombre.subcapa*                    | Crea una capa CSS con *nombre*, con una subcapa anidada *subcapa*. |
  | `@layer` *nombre1*, *nombre2*, *nombre3*...  | Declara y establece un orden para varias capas CSS.          |
  | `@import(`*fichero.css*`) layer(`*nombre*`)` | Importa un archivo CSS y lo introduce en la capa indicada.   |

  El orden en el que se van a aplicar las capas es a medida que van apareciendo (orden en el que han sido definidas). Determina el orden en el que se va a aplicar la especificidad, aunque ese orden se podría alterar.

  - **Capas con nombre:**

    Indicar el nombre a continuación de @layer y añadir el código CSS con los estilos en su interior:

    ```css
    @layer reset {
      body {
        margin: 0;
        box-sizing: border-box;
      }
    }
    ```

    Aquí hemos creado una capa llamada **reset**, que contiene código CSS para resetear ciertas propiedades. El nombre reset, lo establece el desarrollador, y puede ser el que se desee.

  - **Anonimas:**

    Si no ponemos nombre detras de @layer, creamos una capa anónima.

    ```css
    @layer {
      .primary {
        background: #34a;
        border: 2px outset #6381db;
        color: #fff;
        padding: 5px 10px;
        border-radius: 6px;
      }
    }
    ```

    Las capas anónimas, al no tener nombre, no existe ninguna forma de hacer referencia a ellas posteriormente. Si creamos múltiples capas anínimas, el navegador las colocará en capas anónimas diferentes.

  - **Subcapas:**

    Capas anidadas (capas dentro de otras), para ello se utiliza la regla @layer dentro de otra regla @layer o separando los nombres de la capa con puntos:

    Normal

    ```css
    @layer base {
      @layer reset {
        body {
          margin: 0;
          box-sizing: border-box;
        }
      }
    }
    ```

    Resumida

    ```css
    @layer base.reset {
      body {
        margin: 0;
        box-sizing: border-box;
      }
    }
    ```

    

  - **Importaciones:**

    Con la regla @import, importamos código CSS de ficheros externos ea nuestra página. Se puede hace utilizando layer con el nombre de la capa.

    ```css
    @import url("framework.css") layer(framework);
    ```

    Esto permite que código externo se pueda colocar directamente en una capa

  

  Ordenando Capas:

  Las capas se van creando a medida que las escribimos, y mantienen el orden de creación:

  1. Si no tenemos ninguna regla @layer, el navegador mete todo el código en una capa anónima.
  2. Si tenemos reglas @layer, las va creando a medida que las encuentra y mantiene ese orden.

  Sin embargo podemos cambiar el orden de las capas si establecemos una regla @layer con los nombres separados por coma. Es importante que este orden, de definirse, tiene que hacerse antes de crear cualquier otra @layer. Una vez declaradas, no se puede cambiar su orden.

  Veamos un ejemplo, el orden será: primero la capa reset, luego texts y luego theme:

  HTML

  ```css
  <button class="primary">First button</button>
  <button class="primary">Second button</button>
  
  <style>
  @layer reset, texts, theme;
  
  @layer reset {
    button {
      padding: 30px;
    }
  }
  
  @layer theme {
    .primary {
      background: #34a;
      border: 2px outset #6381db;
      color: white;
      font-size: 1rem;
      padding: 5px 10px;
      border-radius: 6px;
    }
  }
  
  @layer texts {
    .primary {
      color: red;
    }
  }
  </style>
  ```

  Centrate en la propiedad **color** de texto de la capa theme y la capa text. Si seguimos el orden natural de CSS, debería aplicarse el color blanco (theme) y luego el color rojo (texts). Sin embargo, como hemos reordenado las capas con la primera línea de @layer, el color blanco prevalece (se aplica después).

  Si en el ejemplo anterior, añadieramos al final el siguiente fragmento de código, el navegador fusionaría las dos capas theme en una misma capa y entonces el color de texto sería gold:

  ```css
  @layer theme{
      .primary{
          color: gold;
      }
  }
  ```

  Ten en cuenta que cualquier estilo declarado sin capa, independientemente del orden de aparición, se agrupará en una capa anónima y se aplicará siempre al final del resto de capas declaradas.

  

  ¿Para que sirven la capas?

  Nos ayudan a poder reorganizar nuestro código y manejar la especifidad de una forma mucho más flexible. Es muy común, que a medida que el código CSS de nuestra página crece, o se incluye código CSS de librerías de terceros o similar. Con esta regla @layer tenemos formas un poco más organizadas de controlar cuando se aplican los estilos.

  En este ejemplo, al ser el mismo selector, se aplica el orden y la herencia, y simplemente **se fusionan los estilos**, sobreescribiendo el último a los anteriores de coincidir las propiedades:

  CSS:

  ```css
  .primary {
      color: red;
  }
  
  .primary {
      background: #34a;
      border: 2px outset #6381db;
      color: #fff;
      padding: 5px 10px;
      border-radius: 6px;
  }
  
  .primary {
      margin: 20px;
      color: gold;
  }
  ```

   HTML

  ```html
  <button class="primary">Click me!</button>
  ```
